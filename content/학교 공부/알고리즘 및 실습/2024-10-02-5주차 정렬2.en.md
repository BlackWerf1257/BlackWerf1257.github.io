## Sorting

### Advanced Sorting Algorithms

#### Types

- On average, it takes $$θ(nlog^n)$$ units of time

- Merge sort
- Quick sort
- Heap sort

#### Merge Sort

- Split the array in half
- Sort each half using a recursive function, then merge them
- Time complexity: $$θ(n log^n)$$

<br>

#### Quick Sort

- Generally offers efficient performance
- Selects a single element from the list and rearranges the elements on either side of it based on its value
  - **Note**: If the pivot element is chosen incorrectly, the runtime may increase to **$$θ(n^2)$$**
- After rearranging the elements, the left and right sides of the pivot element are sorted independently

##### Types

###### 1. Hoare Algorithm

- There are two pointers: pointer i, which starts at the beginning of the array, and pointer j, which starts at the end of the array.
- The left pointer moves to the right until it finds a value greater than the threshold, while the right pointer moves to the left until it finds a value less than the threshold.
- When a pointer stops, the positions of the two values are swapped, and this process is repeated until the pointers cross.

###### 2. The Lomuto Algorithm

- Unlike the Hoare algorithm, i and j increase from left to right.
- If i is greater than the threshold value, it remains unchanged; if it is less than the threshold value, i is incremented and then swapped with j.

##### Related Terms

###### Partition

- Dividing the data into two parts centered on the reference point x

###### Conquest

- Sorting two arrays recursively

###### Merger

- Doing nothing

##### Differences from Merge Sort

- Merge sort involves a merging step, whereas quick sort does not.



#### Hip Alignment

- A sorting algorithm that first converts an array into a heap, then removes elements by repeatedly swapping the root node with the last node of the array
- Similar to selection sort in that it swaps with the last element
- Requires no additional space
  - Merge sort requires additional space

##### Features

- Must use a complete binary tree structure
- All levels except the last one are fully populated
- Every node has a value, which must be greater than or equal to the values of its child nodes (max-heap)
- All levels are sorted from left to right

##### Duration

- Time required to create a heap: **$$θ(n)$$**
- In the worst case, it takes $$O(n log n)$$ time

##### Element Removal

- Target: **the largest element**
- Move the element at position A[n-1] to position A[0], then remove A[n-1] from the set of elements to be processed
  - Swap the last element with the first element
- Since we need to retrieve the last value, the operation takes **$$O(log n)$$** time

##### Sorting by Element

- When an element is removed, the element at A[n-1] moves to the A[0] position, which violates the properties of heap sort.
- To restore the properties of heap sort in this state, the elements must be sorted.
- The elements are sorted by continuously comparing values in the direction of the child node of A[0] with the largest value.
