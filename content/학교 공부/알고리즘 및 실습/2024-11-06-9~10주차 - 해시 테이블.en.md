## B-Tree (Trie)

### Definition

- Disk access unit: block (page)
- A single disk access takes about as long as processing hundreds of thousands of instructions
  - If the search tree resides on disk, it is important to reduce the tree's height
  - By keeping the search tree balanced, the number of disk accesses is reduced

### Features

- Unlike a traditional binary tree, a single node can have multiple child nodes

### Insert

- x: The key to be inserted; if t is the root node of the tree, find the leaf node r where x is to be inserted and insert x into r
  - If an **overflow** occurs at this point, it is handled by a sibling node of r that has available space.
  - If there are **no** nodes with available space, r is split into two parts, and the middle key is passed to the parent node.
  - If an overflow occurs in the parent node p during this process, the overflow is resolved in p.

## Hash Table

### Overview

- Technologies for implementing dictionaries
- Provides the best search performance currently available ($$Θ(1)$$)

### Purpose

- Public-key cryptography algorithms
- Compiler symbol tables
- Database storage schemas
- Data synchronization verification
- Encryption of Internet addresses for privacy protection (Yahoo)
- Services requiring extremely fast response times
  - Searching for information related to 119 emergency calls and call numbers
  - Resident registration system

### The Complexity of Store-and-Retrieve

- **Array**
  - $$O(n) $$‎‎ 
- **Binary Search Tree**
  - On average, $$Θ$$
- **Red-Black Tree**
  - In the worst case, $$Θ(log n)$$
- **Hash Table**
  - On average, $$Θ(1)$$

### Features

- The storage location of a value is determined by the value of the element
- Insertion, search, and deletion can be performed in constant time on average
- Useful for services that require very fast response times
- Does not support operations such as finding the minimum element
  - Since it does not support sorting, ranking is also not possible

### Hash Functions

#### Features

- Elements must be sorted uniformly in the hash table

- The calculation must be simple

  Common methods

  ###### Division

  - $$ h(x) = x \pmod{m} $$ 

    m: Table size

  ###### Multiplication


#### Collision

- A situation where two or more elements are about to be assigned to a single address in a hash table

##### Solution

###### Chaining

- Managing elements with the same address using a single **linked list**
- Requires an additional linked list
- When creating the linked list, newly inserted elements are added to the very front of the list
- **Disadvantages**
  - Requires the structure of a linked list
  - As the length of the chain increases, search time increases to $$O(n)$$
  - The operational cost of a linked list is high

###### Open Addressing

- Resolving collisions within the table space
  - Continuing to generate hash values until free space becomes available
  
- No additional space required

- 3 Ways

  - **Linear probing**

    A method of moving one space at a time until a collision occurs

    When reaching the end of the array, it returns to the starting index and begins searching again

    Formula: $$h_i(x) = (h(x) + i) \pmod{m}$$

    **Disadvantages**

    ​  **Primary Clustering**[^1] is vulnerable

  - **Quadratic probing**

    Formula: $$h_x(x) = h(x) + i^2$$

    ​  i: Number of collisions
 
 **Disadvantages**
 
 - Vulnerable to **Secondary Clustering**[^2]
 
 - **Double Hashing**
 
 A method that uses two hashing functions
  
    **Formula**: 
 
 $$h(x) = x \bmod (table size) $$‎‎ 
 
 $$f(x) = (x \bmod h(x)) + 1 $$‎‎ 
 
 $$h_i(x) = (h(x) + i * f(x)) \bmod (table size) $$‎‎
