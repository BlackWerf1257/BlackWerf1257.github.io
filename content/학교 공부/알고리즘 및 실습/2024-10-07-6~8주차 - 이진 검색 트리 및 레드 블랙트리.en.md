## Binary Search Tree

### Related Terms

- **Record**
  - A storage unit containing all information about an entity
    - Example: A person's record: Social Security number, name, address, cell phone number...

- **Field**
  - A component of a record that represents a specific piece of information

- **Primary Key**
  - A field that uniquely identifies each record
  - May consist of two or more fields

- **Search Tree**
  - Each node contains exactly one key, in accordance with the rules
  - This key can be used to determine the location where a record is stored

<br>

### Features

- Each node in a binary search tree contains the full key value, and the key value must be unique for each node.

- The root node is at the top level, and each node can have **up to two** child nodes

- Each node has a key value greater than that of its left child and less than that of its right child

- The time complexity is determined by the height of the tree

  It has a time complexity of $$ O(height) $$

  When calculated using the formula, $$n = 2^{h+1}-1$$

<br>

### Purpose

- Representation of hierarchical relationships
- Family trees, operating system file systems...

<br>

### Insert

- If the root node of the selected tree does not exist, insert the key at the root position and set the left and right child nodes of that root to `NIL`.
- If the root node of the selected tree exists, compare the key of that node with the key to be inserted; if the latter is smaller, insert it into the left child node; if it is larger, insert it into the right child node.

<br>

### Delete

- Deletions are handled in three different scenarios

- In the case of deletion
  1. If the node to be deleted is a leaf node
  2. If the node to be deleted has exactly one child node
  3. If the node to be deleted has two child nodes

<br>

#### If the node to be deleted is a leaf node

- Simply delete the node

#### When a node has exactly one child node

- Connect the parent and child nodes of the node r to be deleted

#### When a node has two child nodes

- Delete the node s, which is the minimum element in the right subtree of the node r to be deleted, and replace r with s

<br><br>

### Red-Black Tree

- Coloring every node in a binary search tree black or red
  - **However, the tree must satisfy the following red-black properties**

	1. The root must be black
  1. All leaf nodes must be black
  1. If a node is red, its child must be black
  1. The number of black nodes encountered on any path from the root node to any leaf node is the same

#### Insertion in RedBlackTree

- When inserting a new node, it is always inserted as red.
  - When a new node is inserted, if the parent node is black, there is no problem; however, if it is red, property 3 of the red-black tree is violated.
