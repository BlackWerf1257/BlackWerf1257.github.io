## Graph

### Definition

- A representation of phenomena or objects using **vertices** and **edges**

$$Graph G = (V, E) $$‎‎ 

​    V: Set of vertices

E: Set of edges

- Two vertices are said to be **adjacent** if they are connected by an edge

  An edge represents the relationship between two vertices.

- **Application Examples**

- | Graph | Node | Edge     |
  | ------ | ------------ | -------- |
  | Communication | Telephone, Computer | Fiber-optic cable |
  | Circuit | Processor     | Wire |


### Types

#### Adjacency Matrix

- A graph represented as an **N x N** matrix

  (N is the total number of vertices)

  - Element (i, j) = 1: There is an edge between vertices i and j
  - Element (i, j) = 0: There is no edge between vertices i and j

- **For a directed graph**
  - The element (i, j) indicates whether there is an edge connecting vertex i to vertex j

- **For weighted graphs**
  - An edge (i, j) has a **weight** instead of a value of 1

#### Adjacency List

- Representing the graph as N linked lists
- The i-th list contains the vertices adjacent to vertex i
- **For weighted graphs**
  - The list also stores the weights

#### Adjacent Arrays

- Representation using N linked lists
- The i-th list contains the vertices adjacent to vertex i
- **For weighted graphs**
  - Just like an adjacency list, the list also stores the weights

### Minimum Spanning Tree

- **Conditions**
  - Undirected connected graph
    - Connected graph: A graph in which there is a path between every pair of vertices
- **Tree**
  - A connected graph with no cycles
  - A tree with n vertices always has **n-1 edges**
- **Spanning Tree of Graph G**
  - A tree composed solely of the vertices and edges of G
- **Minimum Spanning Tree of G**
  - The spanning tree of G with the minimum total edge length

#### Types

- ##### Kruskal's Algorithm

  - **Principle**:

    - A method that sorts all edges in ascending order by weight and connects them until all vertices are connected

  - **How to perform**:

    1. Initialize n sets, each consisting of a single vertex

    2. Sort the set of edges Q (= E) in ascending order by weight

    3. Select edges from the sorted list starting with the one with the lowest weight, connect the vertices of those edges, and remove the minimum-cost edge (u, v) from Q.

          In this case, if vertices u and v belong to different sets, the two sets must be merged into one

    4. Repeat step 2 until all vertices are connected (while the number of edges in the spanning tree T is less than n-1)

    **However**, you must not select an edge between vertices that are already connected (as this would create a cycle).

  - **Time Complexity**

    - $$O(ElogV)$$

- ##### Prim's Algorithm

  - **Principle**: 

    - A method of expanding to unconnected vertices accessible from a specific vertex

  - Time complexity: 

    Total edge BFS visitation time: $$O(E + V)$$

    Finding the minimum number of edges using the Hip algorithm $$O(log V)$$

    $$ O(E + V) × O(log V)

    ​ = O((E + V) log V)

    ​ = O(ElogV)$$

  - Prim(G, r): A graph G consisting of (V, E), where r is the source vertex

  - How to perform:

    1. Create the empty set S

    2. Mark vertex r as visited and add it to set S.
    3. If S is not V, find the edge (x, y) with the **shortest length** connecting V and S in S, mark vertex y as visited, and add it to set S.
