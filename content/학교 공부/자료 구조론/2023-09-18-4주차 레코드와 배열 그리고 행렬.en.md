## Week 4

### Records

#### Definition

- A structure composed of multiple fields that are related to one another but have different data types and sizes
  - Example: Attendance sheet
- Fields: Each has a different data **type** and **size**

#### Declaration

- Using structures

  ~~~c
  struct Structure-Name{
 data-type member 1;
 .
 .
 .
 data-type member n;
  }
  ~~~

#### Field References

- Used to refer to a specific element within a record
- Uses the field reference operator **.**
  - Example: person.age, car.number

#### Record Array

- An array in which each element is a record structure

#### Nested Records

- When a field within a record is another record
- When a structure is nested within another structure

 <br><br>

### Arrays

#### Definition

- A data structure in which elements of the same data type are arranged in a two-dimensional rectangular grid
- Consists of pairs of **indexes** and **values**

- Is finite

#### One-dimensional array

##### Features

- Vector
- When a two-dimensional array has only one row or one column

##### Methods of Expression

- Declare the array by specifying its range (lower and upper bounds)

##### Calculating the Address of a Specific Element

###### Assumptions

- Array: A(L:U)
- Each element occupies K bytes of memory
- Array start address: B (upper index), L (lower index)

###### Number of Elements

You are one plus one

###### Size of the entire array

- k × (U - L + 1)

###### The address of A(i) at index i

- B + K(i - L)

#### Multidimensional Arrays

##### Features

- Consists of m vectors, each of length n
- Logically, a rectangle comprising rows and columns
- Each element is represented using rows and columns as indices

#### Definition

##### Square Matrix

- A matrix with the same number of rows and columns

###### Notation

- nxn

##### Diagonal Matrix

- A matrix in which all elements other than those on the main diagonal are 0
- Elements on the main diagonal: A11, A22, ..., Ann

##### Sparse matrix

###### Definition

- A matrix with a relatively large number of elements equal to 0

###### Efficient Representation Methods for Sparse Matrices

**Method**

- In the first cell, write the size of the entire matrix and the number of non-zero elements in that order

- Starting from the second cell, write down the row, column, and value of each non-zero element in that order

- Example

  ![Image Alt text]({{link}}/assets/img/DS/4nd/Sparse_Mat.png )

**Advantages**

- Memory savings
- Fast search

**Disadvantages**

- Complex to modify
