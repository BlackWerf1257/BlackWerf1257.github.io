### Stack

#### The Concept of a Stack

* A stack is a type of memory with a single entry point.
  As shown in the image below, data is pushed onto the stack from the top, and the data entered last is removed first.

#### Features of Stack

* Created when the function is called and destroyed after the call.
* The size of the variable cannot be adjusted.
* Allows for fast access.
* Accessible only within the method.
* FILO (First In Last Out)
* LIFO (Last In First Out)

### Queue

#### The Concept of a Queue

* A type of memory with open ends on both sides

* As shown in the image below, data is appended to one end, and when values are removed from memory, the values added first are removed first.

  

#### Features of a Queue

* FIFO (First In First Out)
* A memory area managed directly by the user
* Memory is dynamically allocated and deallocated by the user
* Memory is allocated from lower addresses to higher addresses
* No limit on memory size
* Access is relatively slower than the stack
* Variables can be accessed globally
* Since it is a memory area managed directly by the user, memory management is required

#### Comparison of Binary Search Trees and m-ary Search Trees

- The data stored in a search tree differs only in the values of the keys

  <br>

| | Binary Search Tree | M-ary Search Tree | 4-ary Search Tree | 9-ary Search Tree |
| ------------------------------------------ | ------------- | ------------- | ------------- | ------------- |
| Number of max/min branches | 2 / 0 | M / 0 | 4 / 0 | 9 / 0 |

| Number of nodes required to handle 10 key values | 10 nodes     | | 4 nodes | 2 nodes |

### Hashing

#### Definition

- A method that uses hash **functions** and hash **tables** to enable fast **storage and retrieval** of data

#### How to Use

![Image Alt text]({{link}}/assets/img/DS/2nd/Hashing_Mechanism.png )

#### Advantages

- Search time is always the same regardless of table size, and it is faster than binary search
- Insertions and deletions are easy

#### Disadvantages

- If data with different key values are stored in the same hash table, collisions may occur.
- If the hash table is underutilized, it results in significant memory waste.
