### Stack

#### Definition

- A type of memory where the most recently inserted data is stored first
  - Also known as **L**(ast)**I**(n)**F**(irst)**O**(ut) or a **Pushdown List**
  - **Insertions and deletions are performed at one end**

#### Top Pointer

- A special register used to maintain location information
- Holds the location of the **most recent** entry

#### Implementation Method

##### When the length is known

- Uses a one-dimensional array

#####  Create

- Create an empty stack S

##### Top

- Returns the items at the top of S as the result

##### Push(x, s)

- Inserts element x into S and returns the result of the modified stack

##### Pop(s)

- Remove the top element from S and return the result of the modified stack

##### Empty(s)

- Returns true if S is empty
- Returns false if S is not empty

##### Initial Stack State

  - (s -> top) == 1

#### Multiple Stacks

- Operates n stacks within a single storage block

##### Operating two stacks

- Stack1(top1, bottom1)
- Stack2(top2, bottom2)
- Problems arise when push operations are concentrated on the two stacks
- If top1 = top2, a stack overflow occurs

##### Managing N Stacks

- Each stack contains a top pointer and a bottom pointer.
- Empty state of Stack1: top(i) = bottom(i)
- Overflow state of Stack1: top(i) > bottom(i+1)

### Operator Precedence

- Operations are performed character by character in the order **left to right**
- The order of operations in computers differs from standard arithmetic
- Each programming language defines the order of operations for each operator, and operations are performed in descending order of precedence
- The following is the order of operator precedence

| Operator Precedence |   Operator Type    |
| :-------------: | :--------------: |
| 1 | \|\| |
| 2 | && |
| 3 |   <, <=, >, >=   |
| 4 | +, - |
| 5 |     *, /, % |
| 6 | Unary operators -, ! |

#### When operators with the same precedence exist

- **Outside parentheses**: Evaluation proceeds starting from the leftmost parenthesis
- **Inside parentheses**: When operators with the same precedence appear multiple times
  - Unary operators are evaluated from right to left
  - Non-unary operators are evaluated from left to right

#### How to say it in Polish

- A method for efficiently representing arithmetic expressions

  ###### Classification by Operator Position

  - Infix (infix notation)
    - Structured in the order **operand - operator - operand**
 - Ex)  A +B
  - Prefix (prefix notation)
    - Composed in the order **operand - operator - operand**
 - Ex) +AB
  - Postfix (postfix notation)
    - Composed in the order **operand - operand - operator**
 - Ex) AB+
    - This is the way computers perform calculations
 - The compiler converts infix notation to postfix notation
 - Calculations are performed using a stack

  ###### Infix to Postfix Conversion

  - Move the operator immediately following the closing parenthesis corresponding to that operator, then remove the parenthesis

  ###### Advantages of Postfix

  - No parentheses required
  - No operator precedence required
  - Calculations are easily performed by reading expressions in **left-to-right** order using a stack

  ###### Calculations Using Postfix-Style Stacks

  - Read the elements that make up the expression from left to right
  - If an element is an operand, proceed by pushing it onto the stack
  - If an element is not an operand, proceed by popping the required number of operands from the stack
  - Then, push the result of the operation onto the stack
