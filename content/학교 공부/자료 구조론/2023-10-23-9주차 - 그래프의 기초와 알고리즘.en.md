## Graph

### Orthogonal List

#### Definition

- A simple list structure for representing sparse matrices

- When using this method, it is easy to calculate the outgoing and incoming degrees for any vertex $$V_i$$

- Node types in a list

  ##### Head Node

  - Contains 4 fields

  ###### Field

  - **Flag**
    - Distinguishes between head nodes and edge nodes
  - **Index**
    - Row and column indices of a sparse matrix
  - **Down**
    - Indicates the next non-zero element in the same **column**
  - **Right**
    - Indicates the next non-zero element in the same **row**
 
 ##### Edge Node

  - Contains 5 nodes

  ###### Field

  - **Flag, Right, Down**
    - Represents the next non-zero element in the row
  - **Tail**
    - The tail of the edge
  - **Head**
    - The head of the edge

### Graph Traversal

#### Definition

- Given an undirected graph $$G=(V, E)$$ and a vertex \(V\) in \(V(G)\), starting from vertex \(V\), visit every vertex in the graph exactly once 

  Visiting every vertex

####  Types

##### Depth-First Search (DFS)

###### How to Perform

1) Determine and visit the starting vertex **v**<br>
2) Select an unvisited vertex **w** adjacent to the starting vertex **v** and begin a DFS search<br>
3) If no further progress is possible, return to the last branching point and continue from there<br>
4) If there are no more nodes to visit, terminate the search

##### Breadth-First Search (BFS)

###### How to Perform

1. Start at the initial vertex **v**<br>
2. Visit all vertices **w** adjacent to **v** first<br>
3. After visiting all vertices **w**, visit all vertices adjacent to **w** that have not yet been visited<br>
4. Continue the search until there are no more nodes to visit

<br>

### Kidney Tree

#### Kidney Tree

##### Definition

- A tree[^1] that contains some or all of the edges of a graph $$G$$ and all of its vertices

  [^1]: There are no cycles between any two vertices

- A subgraph $$G'$$ of the graph $$G$$

##### Types

###### Depth-First Spanning Tree

- A sparing tree generated by DFS

###### Breadth-First Spanning Tree

- A spanning tree generated by BFS

##### Applications

###### Road Network Construction Plan

- Problem Definition: **Connecting all cities via the shortest paths**
  - Assign weights or distance values to each edge
  - To solve the problem, the total cost must be minimized
- **Cost of a spanning tree**
  - The sum of the weights of all edges in the tree

<br>

#### Minimum-Cost Spanning Tree

##### Definition

- A spanning tree that minimizes the total weight when each edge of graph G is assigned a weight of +

##### Types

###### Prim Algorithsm

**Definition**

- Select the edge with the lowest cost and include it in the minimum-spanning tree T
  - Among the edges connected to the vertices in tree T, include the edge with the lowest cost in tree T
- If a cycle is created, discard the current progress and select the edge with the lowest cost from the unvisited edges
- The search terminates once all vertices have been visited

###### Kruskal Algorithsm

**Definition**

- Select the edge with the lowest weight among all edges
  - If the selected edge creates a cycle, **exclude** it
- Repeat the above process until all vertices are connected

###### Sollin Algorithm

**Definition**

- Select one minimum-cost edge for each vertex in the graph
- Select one minimum-cost edge for each tree within the forest
- Continue the search until there are no more edges or a complete tree is formed

<br>

**Differences Between Prim's and Kruskal's Algorithms**

- The Prim and Kruskal algorithms select one edge at each step to search
  - The Sollin algorithm selects multiple edges
