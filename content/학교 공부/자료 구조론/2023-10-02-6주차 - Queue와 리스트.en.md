## Course Content

### Queue

#### Definition

- A memory structure where items are inserted at one end and removed at the other

![Image Alt 텍스트]({{link}}/assets/img/DS/2nd/Queue.png )

- **F**(irst)**I**(n)**F**(irst)**O**(ut)

#### Operations on Queues

- The operation proceeds by incrementing the value of the corresponding pointer by 1
  - Insertion: Front Pointer
  - Deletion: Rear Pointer
- A logical deletion is performed

#### Implementation Method

- Array
  - Implemented as a one-dimensional array
    - Q = [Q(0), ..., Q(t-1)]
    - **Front(Q)**: The data at the position pointed to by the queue's front pointer
    - **Rear(Q)**: The data at the position pointed to by the queue's rear pointer
    - **Size(Q)**: The number of data items in the queue
- Linked List

#### How to Declare

- Declared using a struct
- A queue must contain three members:
  1. An array of **finite** size to store items
  2. An integer variable for the front pointer
  3. An integer variable for the rear pointer

#### Operations for Data Structure Applications in C

- **Create**: Create an empty queue
- **Insert(x, Q)**: Insert element x at the position pointed to by the **Rear Pointer**, then return a new queue
 
 - Result: rear(Insert(x, Q)) = s;
  - Overflow must be considered during insertion
- **Delete(Q)**: Deletes the item at the position pointed to by the **Front Pointer** and returns a new queue

  - Underflow must be considered when deleting elements
- **Front(Q)**: Returns the front element of the queue
- **Empty(Q)**: Returns whether the queue is empty

  - Returns True if the queue is empty
  - Returns False if the queue is not empty

  <br>

  - The insertion and deletion algorithms in a queue move from left to right as time progresses

  #### Issues

  ##### 1. When the rear pointer points to the last element of the array (q->r == n-1)

  - You must specify n items in the queue

    Unnecessary overflow may occur

  - This results in reduced memory efficiency

  ##### When Unnecessary Overflow Occurs

  ###### Front Pointer > -1

  If (q->f) > -1, q => queue0[], ..., q->queue[q->f] represents available free space

  ###### When deleting consecutive items from a full queue

  -> An overflow occurs when the variable is empty

#### Solution

##### Rearrange the items in the queue

- Shift the entire queue to the left so that the first element of the queue, q->queue[q->f+1], is positioned at q->queue[0]
- This can be achieved by setting the value of the front pointer q->f to -1 and adjusting the value of q->r

###### Issues

- Requires reordering operations
  - Processing time is proportional to the number of items in the queue

#### Circular Queue

- A queue that represents an array in a circular structure rather than a linear one
- A structure where the **beginning and end of the array are connected**
- An example of a circular queue

![Image Alt 텍스트]({{link}}/assets/img/DS/6nd/Circle_Queue.png )

##### Operations on a Circular Queue

- Increases the Front Pointer or Rear Pointer by 1 in a clockwise direction

###### Insert Item

- Use the remaining modulo operator to calculate the new rear pointer
  - Example: rear = (rear + 1) % n
- **Important Note**
  - An overflow may occur if (q->f) == (q->r)

###### Delete Item

- A new front pointer must be calculated

  - Can be calculated using the modulo operator

    **front = (front + 1) % n**

- **Important Notes**

  - If (q->f) == (q->r), an underflow may occur

### List

#### Sequential (Linear) List

##### Definition

- A list that stores each data item sequentially in contiguous memory

- A collection of data elements in which a(i) precedes a(i+1)

  (Example): **List of dates**

#### Sequential Mapping

##### Definition

- When implementing a sequence list as an array, the state in which the i-th data item in the sequence list is stored in the i-th element of the array

##### Advantages

- Enables efficient use of memory

##### Disadvantages

- When data is inserted or deleted, it is necessary to preserve the order of the data in the list

##### Issues with Data Deletion

- In a list consisting of n elements, if the kth element is deleted, the number of data movements is **n-k**.

##### Issues When Inserting Data

- In a list consisting of n elements, inserting the kth element results in **n-k+1** element swaps.

#### Singly Linked List

##### Definition

- A non-sequential mapping method that physically distributes logically related data across memory
- Data is represented as nodes, and each node contains two fields

###### Data Fields

- The role of storing the data being processed

###### Link Field

- The role of storing the address or location information of the next node

<br>

- A set of connected nodes; link information is used to maintain data order

  - Link value of the last node: **NULL** 

  ![Image Alt text]({{link}}/assets/img/DS/6nd/Simple_Conn_List.png )  

##### Physical Structure

- Data is **not stored in sequential order** within memory
- Data order is maintained using **link fields**
