## 授業内容

- SQLiteとの連携、データの追加および値の取得

SQLiteとは何ですか？

- 公式サイトの説明によると、SQLiteは小型かつ高速で、スタンドアロン型であり、信頼性と完全な機能を備えたC言語ベースのSQLエンジンライブラリである。
- Androidでは、組み込みデータベースとしてSQLiteが使用されている。

<br/>

### SQLiteの仕組み

#### SQLiteOpenHelper クラス

- DBおよびテーブルの作成
- 継承後に再定義して使用

#### SQLiteDatabase クラス

- SQL文の実行用

#### Cursor インターフェース

- カーソル移動用
  - カーソル：データベースに保存されたテーブルの行を参照し、データを取得する役割を果たす

<br/>

### 主なメソッド

| クラスまたはインターフェース  | メソッド名 | 用途 |
| ----------------------- | --------------------- | ----------------------------------------------------- |
| SQLiteOpenHelper クラス | コンストラクタ | DBの作成 |
| | onCreate() | テーブルの作成 |
| | onUpgrade() | テーブルの削除および再作成 |
| | getReadableDatabase() | 読み取り専用モードでDBを開く<br>SQLiteDatabaseを返す |
| | getWritableDatabase() | 読み書き専用モードでDBを開く<br/>SQLiteDatabaseを返す |
| SQLiteDatabaseクラス   | execSQL() | SQL文（Insert、Update、Delete）を実行 |
| | close() | DBを閉じる |
| | query()<br>rawQuery() | Selectを実行し、カーソルを返す |
| Cursorインターフェース | moveToFirst() | カーソルを最初の行に移動 |
| | moveToLast() | カーソルを最後の行に移動 |
| | moveToNext() | 現在のカーソルの次の行に移動 |

<br/>

### 使用したコード

#### DB処理用クラス

~~~java
public class DBClass extends SQLiteOpenHelper{
 public DBClass(Context context){ //データベースの作成
 super(context, "groupDB", null, 1);
 }
  @override
    public void onCreate(SQLiteDatabase db){ //テーブル作成
 db.execSQL("Create Table testTable(gName Char(20) Primary Key, gNum Integer)");
    }
    @override
    public void onUpgrade(SQLiteDatabase db, int oldVersion, int newVersion){
 //同じテーブルが存在する場合、削除して再作成する
 db.execSQL("Drop Table IF Exists testTable");
 onCreate(db);
    }
}
~~~

- コンストラクタと必須メソッド [**onCreate()**, **onUpgrade()**] が必要である
- Code - Override Methods を使用してコードを読み込むことができる

<br>

##### データベースの作成

- DB処理用クラスのコンストラクタを使用して、データベースを作成することができる
- パラメータとして、@Nullable Context、@Nullable String、@Nullable SQLiteDatabase.CursorFactory、int を持つ
- 各パラメータの意味は以下の通りである
  - context: データベースへのパスを特定するために使用される
  - name: 作成するデータベースの名前を意味する
  - factory: カーソルオブジェクトを作成したり、デフォルト値をnullに指定したりするために使用される
  - version: データベースのバージョンを意味し（1から始まる）、**onUpgrade**や**onDowngrade**を利用して値を調整できる

##### テーブルの作成

- **onCreate** を使用してテーブルを作成することができる

  - パラメータとしてSQLiteDatabaseを持つ
  - onCreate関数内に**db.execSQL("Create Table テーブル名(変数名 データ型 primary key, 変数n データ型);");**を記述することで、テーブルを作成できる
  - 例

  ~~~java
  @Override
 public void onCreate(SQLiteDatabase db) { //テーブルの作成
              //引数 SQLiteDatabase db execSQL("クエリ")
 //テーブル作成クエリを入力するクエリ文
 db.execSQL("Create Table groupTBL(gName CHAR(20) Primary Key, gNumber Integer);");
 } //onCreate
  ~~~

##### テーブルの初期化

- テーブルの初期化方法は簡単です
- **onUpgrade**または**onDowngrade**でテーブルの有無を確認した後、削除およびテーブル生成関数を呼び出せばよい
- テーブルの有無は、次のようなクエリを実行することで確認できます

~~~java
db.execSQL("Drop Table IF Exists テーブル名");
~~~
