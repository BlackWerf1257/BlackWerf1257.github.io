## Course Content

- Integrating with SQLite, adding data, and retrieving values

What is SQLite?

- According to the official website, SQLite is a small, fast, self-contained, reliable, and fully featured C-based SQL engine library.
- Android uses SQLite as its built-in database.

<br/>

### How SQLite Works

#### The SQLiteOpenHelper Class

- Used for creating databases and tables
- Redefined and used after inheritance

#### The SQLiteDatabase Class

- For executing SQL statements

#### Cursor Interface

- For cursor navigation
  - Cursor: Retrieves data by referencing rows in a table stored in the database

<br/>

### Key Methods

| Class or Interface  | Method Name | Purpose |
| ----------------------- | --------------------- | ----------------------------------------------------- |
| SQLiteOpenHelper Class | Constructor | Create DB |
| | onCreate() | Create tables |
| | onUpgrade() | Remove and recreate tables |
| | getReadableDatabase() | Open the database in read-only mode <br> Return an SQLiteDatabase |
| | getWritableDatabase() | Open the database in read-write mode <br/> Return an SQLiteDatabase |
| SQLiteDatabase Class   | execSQL() | Execute SQL statements (Insert, Update, Delete) |
| | close() | Close the database |
| | query()<br>rawQuery() | Execute SELECT and return a cursor |
| Cursor interface | moveToFirst() | Move to the first row of the cursor |
| | moveToLast() | Move to the last row of the cursor |
| | moveToNext() | Move to the next row of the current cursor |

<br/>

### Code Used

#### Classes for Database Processing

~~~java
public class DBClass extends SQLiteOpenHelper{
 public DBClass(Conetext context){ //Create database
 super(context, "groupDB", null, 1);
 }
  @override
    public void onCreate(SQLiteDatabase db){ //Table Create
 db.execSQL("Create Table testTable(gName Char(20) Primary Key, gNum Integer)");
    }
    @override
    public void onUpgrade(SQLiteDatabase db, int oldVersion, int newVersion){
 //If same table exists, deletes and recreate it
 db.execSQL("Drop Table IF Exists testTable");
 onCreate(db);
    }
}
~~~

- A constructor and the required methods [**onCreate()**, **onUpgrade()**] are required
- You can load code using the "Code - Override Methods" feature

<br>

##### Creating a Database

- You can create a database using the constructor of the DB handling class.
- It takes the following parameters: @Nullable Context, @Nullable String, @Nullable SQLiteDatabase.CursorFactory, and int.
- The meaning of each parameter is as follows
  - context: Used to locate the path to the database
  - name: Represents the name of the database to be created
  - factory: Used to create a cursor object or set the default value to null
  - version: Represents the database version number (starting from 1); this value can be adjusted using **onUpgrade** or **onDowngrade**

##### Creating a Table

- You can create a table using **onCreate**

  - Takes an SQLiteDatabase as a parameter
  - You can create a table by including **db.execSQL("Create Table table_name(variable_name variable_type primary key, variable_n variable_type);")** in the onCreate function
  - Example

  ~~~java
  @Override
 public void onCreate(SQLiteDatabase db) { // Create table
              //Parameter: SQLiteDatabase db execSQL("query")
 //Enter the table creation query
 db.execSQL("Create Table groupTBL(gName CHAR(20) Primary Key, gNumber Integer);");
 } //onCreate
  ~~~

##### Initialize Table

- Initializing a table is simple.
- In **onUpgrade** or **onDowngrade**, check whether the table exists, and if not, call the delete and create table functions.
- You can check whether the table exists by executing the following query:

~~~java
db.execSQL("Drop Table IF Exists 테이블명");
~~~
