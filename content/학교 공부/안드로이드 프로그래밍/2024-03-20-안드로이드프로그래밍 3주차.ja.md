## 授業内容

### Androidの主な機能

- アプリケーションフレームワークが提供するAPIを利用することで、

  コードの再利用により、効率的で迅速なアプリ開発が可能である

- DalvikやART（Android Runtime）を提供する

  - Dalvik：AndroidでJavaアプリを実行するためのJIT方式のバイトコードコンパイルモード

    Lollipop以降のバージョンではARTに置き換えられた

  - Artランタイム：Android Runtimeの略称で、Dalvikの限界を克服するために開発された

- モバイル用データベースであるSQLiteを提供しています

- 各種オーディオ、ビデオ、画像の形式に対応しています

- BluetoothやWi-Fi、コンパスなど、さまざまなハードウェアに対応しています

- Eclipse IDEやAndroid Studioを活用した迅速な開発を可能にする

<br>

### Androidの特徴

- コアカーネルはLinuxで構成されている
- アプリの開発言語はJavaである
- Android SDKには多くのライブラリが含まれており、開発が容易である
- オープンソースを重視している
- 継続的かつ迅速なアップデートを提供している

<br/>

### Androidの構造

- #### アプリケーション

  - Androidアプリで利用可能な一般的なアプリケーション
  - ブラウザ、カレンダーなど、ユーザーにとって最もよく使われるもの
  - Javaで記述されている
  - 種類：ホーム、連絡先、電話、ブラウザなど……

- #### アプリケーションフレームワーク

  - Android APIが存在する
  - ハードウェアにアクセスする際は、APIを介してのみアクセスが可能である
  - 種類：Activity Manager、Window Manager、Content Providers、View System、Package Manager、Telephony Manager、Resource Manager、Location Manager、Notification Manager

- #### ライブラリ

  - Androidのシステムライブラリは、システムへのアクセスを目的としてC言語で記述されているため、高性能であり、きめ細かな操作が可能である

  - 種類: Surface Manager、Media Framework、SQLite、

    OpenGL ES、FreeType、WebKit、SGL、SSL、libc

- #### Androidランタイム

  - JavaコアライブラリとDalvik仮想マシンまたはARTランタイムで構成されている
  - AndroidはJavaでプログラミングされるが、Java仮想マシンの代わりにDalvik仮想マシンまたはARTランタイムを利用している
  - 種類：Core Libraries、Dalvik VM

- #### Linuxカーネル

  - ハードウェアの制御に関連する管理機能が含まれている

    - メモリ管理、デバイスドライバ、セキュリティなど……

  - 多くのスマートフォン内のデバイスに対応しています

    - カメラ、タッチスクリーン、GPS、ジャイロスコープなど……

  - 種類：ディスプレイドライバ、カメラドライバ、フラッシュメモリドライバ、バインダードライバ（IPC）、キーパッドドライバ、Wi-Fiドライバ、

    オーディオドライバ、電源管理

### Android Studioで壁紙を追加する

#### 背景を色で塗りつぶす

- **android:background="#RGBコード値"> **を使用して利用可能です

- RGBコード値は、手動で入力する方法と、あらかじめ登録しておいて読み込む方法がある

##### 手動入力

RGBコードの値に「FF0000」のように直接数値を入力して指定することができます

##### colorを登録して使用する

- プロジェクト内の「values」フォルダに「colors.xml」ファイルが存在する

- ~~~c++
  <?xml version="1.0" encoding="utf-8"?>
  <resources>
 <color name="black">#FF000000</color>
 <color name="white">#FFFFFFFF</color>
  </resources>
  ~~~

  colors.xml ファイル内では、次のように色が指定されている。

  ​  blackとwhiteの例のように色を追加すると、その色が保存されます

  この色を呼び出すには、以下の方法を利用すれば可能です

- 色をインポート

  **color/色名**

  例）color/black

### Android StudioでImageViewを使用して写真を追加する

#### 実装方法

##### 方法 1

1. resフォルダの下にあるdrawableフォルダに、使用する画像を配置する
2. activity_main.xmlファイルにImageViewを追加する
3. MainActivity.javaファイルでImageViewを宣言する
4. **ImageView変数名.setImageResource(R.drawable.ファイル名);** を使用してファイルを選択する

使用したコード

**activity_main.xml**

~~~xml
<?xml version="1.0" encoding="utf-8"?>
<LinearLayout xmlns:android="http://schemas.android.com/apk/res/android"
    xmlns:app="http://schemas.android.com/apk/res-auto"
    xmlns:tools="http://schemas.android.com/tools"
    android:layout_width="match_parent"
    android:layout_height="match_parent"
    tools:context=".MainActivity"
    android:orientation="vertical"
    android:background="@drawable/test_img">

    <ImageView
 android:id="@+id/imgView"
 android:layout_width="match_parent"
 android:layout_height="wrap_content"></ImageView>
</LinearLayout>
~~~

**MainActivity.java**

~~~java
package com.example.background20214135;

import androidx.appcompat.app.AppCompatActivity;

import android.os.Bundle;
import android.widget.ImageView;

public class MainActivity extends AppCompatActivity {
    ImageView imgView;
    @Override

 imgView.setImageResource(R.drawable.pngegg);
    }
}
~~~

### Android Studioでの画面移動

#### 実装内容

- メイン画面でボタンを押すと、「Second」という2番目の画面に移動する

- 2番目の画面で「2番目の画面のメインへ」というボタンを押すと、1番目の画面に移動する

#### 実装方法

- Android Studioで別の画面に切り替えるには、**Intent**を使用する

  - Intentはアクティビティ間のデータ転送に使用されるクラスであり、画面遷移だけでなく、Listなどのデータも転送することが可能です

- 以下のコードを使用すると、画面の生成や移動が可能になります

  ~~~java
  Intent オブジェクト名 = new Intent(取得するデータ情報, 遷移先の画面);
  startActivity(オブジェクト名);
  ~~~

- 別の画面から元の画面に戻る際は、Javaファイルを終了させるコマンドとして**finish();**を使用すればよい。

##### 注意事項

- 別の画面を作成するには、**java**および**xml**形式のファイルを作成する必要がある

<br>

#### 使用したコード

**activity_main.xml**

~~~xml

    android:layout_width="match_parent"
    android:layout_height="match_parent"
    android:orientation="vertical"
    tools:context=".MainActivity"
    >

    <Button
 android:id="@+id/mainBtn1"
 android:layout_width="match_parent"
 android:layout_height="wrap_content"

 />
</LinearLayout>
~~~

**second.xml**

~~~xml
<?xml version="1.0" encoding="utf-8"?>
<LinearLayout xmlns:android="http://schemas.android.com/apk/res/android"
    android:orientation="vertical"
    android:layout_width="match_parent"
    android:layout_height="match_parent"
    android:background="#FF0000">

    <Button
 android:id="@+id/subB1"
 android:layout_width="match_parent"
 android:layout_height="wrap_content"
 android:text="2番目の画面のメインへ移動"
 android:textSize="30sp" />

    <!-- Second.java 実際のJava画面の生成 -->

</LinearLayout>
~~~

**MainActivity.java**

~~~java
package kr.ac.baekseok.activity20214135;

import androidx.appcompat.app.AppCompatActivity;

import android.content.Intent;
import android.os.Bundle;
import android.view.View;
import android.widget.Button;

public class MainActivity extends AppCompatActivity {
    Button mBtn1;

    @Override
    protected void onCreate(Bundle savedInstanceState) {
    super.onCreate(savedInstanceState);
    setContentView(R.layout.activity_main);

        mBtn1 = (Button)findViewById(R.id.mainBtn1);

        mBtn1.setOnClickListener(new View.OnClickListener() {
 @Override
 public void onClick(View v) {
 //メイン画面から2番目の画面へ遷移する（second.xml、second.java）
 //インスタンスを作成する

                //現在のアプリの情報、遷移先の画面の情報
 Intent intent = new Intent(getApplicationContext(), Second.class); //コンストラクタ（クラス名と同じメソッド名）
 startActivity(intent); //インテント情報を使用して画面を起動する
 }
 });//mBtn1
    } //onCreate
}
~~~

Intentクラスのコンストラクタを使用して、現在のアプリの情報と遷移先の画面の情報を引数として指定してオブジェクトを作成し、startActivityを使用して作成された画面を実行する。

<br/>

**Second.java**

~~~java
package kr.ac.baekseok.activity20214135;

import android.app.Activity;
import android.content.Intent;
import android.os.Bundle;
import android.view.View;
import android.widget.Button;

import androidx.annotation.Nullable;

public class Second extends Activity {
    // 画面クラスには必須メソッド onCreate() が必要 -> [code] - [override]
    Button secondBtn1;
 
 @Override
    protected void onCreate(@Nullable Bundle savedInstanceState) {
 super.onCreate(savedInstanceState);
 setContentView(R.layout.second); //レイアウトを設定
 
 secondBtn1 = (Button)findViewById(R.id.subB1);
        secondBtn1.setOnClickListener(new View.OnClickListener() {
 @Override
 public void onClick(View v) {
 //メイン画面へ移動（Intentクラス）
 finish(); //Second.javaを終了
 }
 });
    }
} //second

~~~

画面の閉じ処理は、画面遷移の際にMainActivity.javaでインスタンス化して行うのとは異なり、単なる終了であるため、finish()を使用して終了させる。
