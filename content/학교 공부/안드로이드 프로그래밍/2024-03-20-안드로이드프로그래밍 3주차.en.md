## Course Content

### Key Features of Android

- By using the APIs provided by the application framework,

  This enables efficient and rapid app development through code reuse

- Supports Dalvik or ART

  - Dalvik: A JIT-based bytecode compilation mode for running Java apps on Android

    In versions after Lollipop, it was replaced by ART

  - Art Runtime: An abbreviation for Android Runtime, developed to address the limitations of Dalvik

- Provides SQLite, a mobile database

- Supports various audio, video, and image formats

- Supports a variety of hardware, such as Bluetooth, Wi-Fi, and a compass

- Enables rapid development using the Eclipse IDE and Android Studio

<br>

### Features of Android

- The core kernel is based on Linux
- Apps are developed using Java
- The Android SDK includes a wide range of libraries, making development convenient
- Open-source oriented
- Provides continuous and rapid updates

<br/>

### The Structure of Android

- #### Applications

  - Common applications available on Android devices
  - Most frequently used by users, such as browsers and calendars
  - Written in Java
  - Types: Home, Contacts, Phone, Browser, etc....

- #### Application Framework

  - Android APIs exist
  - Access to hardware is only possible through these APIs
  - Types: Activity Manager, Window Manager, Content Providers, View System, Package Manager, Telephony Manager, Resource Manager, Location Manager, Notification Manager

- #### Library

  - The system libraries within Android are written in C to facilitate system access, offering excellent performance and allowing for fine-grained control.

  - 종류: Surface Manager, Media Framework, SQLite, 

    OpenGL ES, FreeType, WebKit, SGL, SSL, libc

- #### Android Runtime

  - Consists of the Java Core Libraries and the Dalvik Virtual Machine or ART Runtime
  - Android is programmed in Java, but uses the Dalvik Virtual Machine or ART Runtime instead of the Java Virtual Machine
  - Types: Core Libraries, Dalvik Virtual Machine

- #### Linux Kernel

  - Includes management functions related to hardware control

    - Memory management, device drivers, security, etc....

  - Supports a wide range of devices, including many smartphones

    - Camera, touchscreen, GPS, gyroscope, etc....

  - 종류: Display Driver, Camera Driver, Flash Memory Driver, Binder Driver(IPC), Keypad Driver, Wifi Driver, 

    Audio Driver, Power Management

### Adding a Background Image in Android Studio

#### Fill the Background with Color

- You can use **android:background="#RGB code value"> **

- There are two ways to enter RGB code values: you can either enter them manually or load them from a pre-registered list.

##### Fill in manually

You can specify the RGB code by entering a value directly, such as "FF0000"

##### Registering and Using Colors

- There is a colors.xml file in the "values" folder within the project

- ~~~c++
  <?xml version="1.0" encoding="utf-8"?>
  <resources>
 <color name="black">#FF000000</color>
 <color name="white">#FFFFFFFF</color>
  </resources>
  ~~~

  Colors are defined in the colors.xml file as follows.

  If you add a color as shown in the examples for black and white, that color will be saved.

  You can load this color using the method described below.

- Load Colors

  **Color/Color Name**

  Example) color/black

~~~java
package com.example.background20214135;
~~~

### Navigating the Interface in Android Studio

#### Implementation Details

- When you tap the button on the main screen, you are taken to the second screen, labeled "Second."

- On the second screen, tapping the "Go to Main Screen" button will take you back to the first screen.

#### Implementation Method

- To navigate to another screen in Android Studio, use an **Intent**

  - An Intent is a class used to pass data between activities; it is not limited to screen transitions but can also be used to pass data such as lists.

- You can create and navigate screens using the code below

  ~~~java
  Intent objectName = new Intent(dataToRetrieve, destinationScreen);
  startActivity(objectName);
  ~~~

- To return to the original screen from another screen, use the command **finish();** to close the Java file.

##### Important Notes

- To create another screen, you must create files in **Java** and **XML** formats.

**MainActivity.java**

~~~java
package kr.ac.baekseok.activity20214135;

import androidx.appcompat.app.AppCompatActivity;

import android.content.Intent;
import android.os.Bundle;
import android.view.View;
import android.widget.Button;

public class MainActivity extends AppCompatActivity {
    Button mBtn1;

    @Override
    protected void onCreate(Bundle savedInstanceState) {
 super.onCreate(savedInstanceState);
 setContentView(R.layout.activity_main);

        mBtn1 = (Button)findViewById(R.id.mainBtn1);

        mBtn1.setOnClickListener(new View.OnClickListener() {
 @Override
 public void onClick(View v) {
 // Navigate from the main screen to the second screen (second.xml, second.java)
 // Create an instance

                //Current app information and data for the target screen
 Intent intent = new Intent(getApplicationContext(), Second.class); //Constructor (method name matches class name)
 startActivity(intent); //Launch the screen using the intent
 }
 });//mBtn1
    } //onCreate
}
~~~

Use the constructor of the Intent class to create an object by passing the current app's information and the information of the screen to be navigated as parameters, and then use startActivity to launch the created screen.
