## プラットフォーム

- ソフトウェアアプリケーションの実行に必要なハードウェアとソフトウェアの集合

### ソフトウェアプラットフォーム

- 同一カテゴリのプログラムが開発・実行できるよう、その間を取り持つソフトウェア環境

- 例）

  Java仮想マシン、ActionScript、.NETプラットフォーム

<br/>

#### .NETプラットフォーム

- マイクロソフト社の中間レイヤーであり、プラットフォームの基本的な発展形態である
- 当初はWindowsでのみ動作したが、現在はマイクロソフト社が対応するすべての言語と連携可能である
- C#は「Mono」というソフトウェアプラットフォーム上でも動作する

<br/>

#### ゲームフレームワーク（ゲームエンジン）

## C#の基本構造

- ヘッダーがない
  - クラス定義や宣言部が存在しない
    - Javaのようにクラス実装部のみが存在する

<br/>

### クラス

- 現実世界にある、プログラミングの対象となる事物をまとめたツール
- メンバー変数とメンバー関数で構成されている
- namespace キーワードを用いてブロックで構成されている
- using を使用してクラスをインポートすることができる
- public アクセス指定：他のクラスからそのクラスを使用できるようにすること

<br/>

### オブジェクト

- 生成される際、クラスの構造そのままの形でメモリ上に生成される
- メンバ変数と関数で構成されており、インスタンスとも呼ばれる
- オブジェクトはそれぞれ独立した領域に生成される
- 生成方法）
  - クラス名 変数（オブジェクト名）；
  - クラス名 変数 = new クラス名();

<br/>

### メンバー

#### 構成要素

- フィールド（メンバー変数）、メソッド（メンバー関数）
- public アクセス指定：他のすべてのクラスからのアクセスを許可

<br/>
