## Static

- ![Image Alt Text]({{link}}/assets/img/OOP/10th/Memory_Structure.png )
- Classified as **static classes**, **static methods**, and **static member functions** depending on their usage

- The heap refers to the area of memory that is dynamically allocated using the `new` keyword.

- Variables stored in the stack area include local variables and parameters.
- The BSS area is where uninitialized global variables and static variables are located; all values are initialized to 0.
- Static variables are located in the data area; they are loaded when the program starts and their memory is released when the program ends.
- There is only one such variable in the program.

<br/>

- Variables managed by a class; classes without the `static` keyword and member variables are managed by objects.
- Depending on where they are declared, all subcomponents must be static.

- After use, the garbage collector handles the stack and heap
 - BSS is X

<br>

### Memory Areas

#### Data Area

- The memory space allocated for static variables
- Allocated when the program starts and released from memory when the program ends

#### Stack Area

- The space where local variables and parameters are stored when a function is called
- It is destroyed after the function call
- Memory is released starting with the variables of the last function called

#### Hip Area

- A memory area managed by the programmer
- A memory area created through dynamic allocation
- Memory is released starting with the first block allocated





<br/>

### Static Class

- Instances cannot be created using the `new` keyword
- All members must be declared as static
- Constructors cannot be included (static members cannot be instantiated using `new`)

- Created on the first call to the object and destroyed when the program exits
  - Static classes are accessible from anywhere
- Useful when creating utility classes that are accessible globally

<br/>

**Example**



```c#
public static class Calculator
{
    public static int Plus(int a, int b) { return a + b; }
    public static int Minus(int a, int b) { return a - b; }
}

public class Program
{
    public static void Main(string[] args)
    {
        int addResult = Calculator.Plus(5, 3);
        int minusResult = Calculator.Minus(5, 3);

        Console.WriteLine(addResult);
        Console.WriteLine(minusResult);
    }
}
```

```
결과 :
5
2
```

### Static Fields (Methods)

- Accessible without creating an instance
- Declared by placing the `static` keyword before the data type
- Accessible from anywhere
  - Primarily used for variables that need to be accessible globally
- Stored in the static memory area → Remains in memory until the program terminates
- Only one value is held per class, regardless of how many are created
- Reason for use: To share values among instances



```c#
public class Book
{
    public static int count = 0;
    private string name;
    public Book(string name) {
        this.name = name;
        count++;
    }
}


public class Program
{
    public static void Main(string[] args)
    {
        //정적 필드
        Book book1 = new Book("Harry Potter");
        Book book2 = new Book("The Lord of the Rings");
        Console.WriteLine("Created Book Count : " + Book.count);
    }
}
```

```
출력 :
Created Book Count : 2
```

  - Therefore, even if multiple instances are created, only one instance exists within the class; thus, when book1 increments by 1 and book2 increments by 1, the value becomes 2
  - If `count` is not declared as `static`, a separate instance of `count` is created for each instance, so the value is 0
