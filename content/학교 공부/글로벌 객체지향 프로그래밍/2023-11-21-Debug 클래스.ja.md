## デバッグクラス

- C#で開発中に何かを出力する場合は、Console.WriteLine() または Console.Write() を使用する
  - しかし、このデバッグ機能はデプロイ後も動作するため、メモリを消費する



- デバッグが過剰な場合の課題

  - 配布時には、このコマンドをすべて削除する必要があります

  - 削除作業中に、他のコードも誤って削除してしまう可能性がある

    →条件付きコンパイル文を使用することで解決可能である



**例**

```c#
#define RELEASE
    class LOG{
        #if RELEASE
            public statoc void print(string str)
	        { Debug.Log(str); }
        #else
            public static void print(string str) { 				//Debug.log(); 
        	}
        #endif
    }
```



- 例のようにデバッグクラスを使用する場合、

  **#define RELEASE** を使用した場合にのみ、Debugが実行される
