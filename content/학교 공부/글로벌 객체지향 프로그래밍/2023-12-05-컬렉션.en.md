## Arrays

### Concept

- Can store data of the same data type
- Data can be inserted or modified by accessing it via an index

<br/>

## Collection

### Concept

- Can store various types of data
- Available by declaring the **System.Collections** namespace

<br/>

### Types

#### ArrayList

- An array of objects whose size dynamically increases as needed
- A collection similar to an array, allowing access to values via an index and enabling direct reading and writing
- Can store values of any type
- Implicit type conversion is possible when retrieving values using the `var` keyword
- Supports adding, removing, modifying, and querying values

<br/>

- ##### Differences from arrays

  - The size of an array is specified when it is created

  - The size of an ArrayList changes dynamically as values are added or removed after it is created.

    

    

    | Method |    Property    | Description |
    | :--------------: | :--------: | :----------------------------------------------------------: |
    |     Add(value) | Store (Add) | Adds a value to the end of the ArrayList |
    | Insert(index, value) | Store (Add) | Adds a value at the specified index<br/>All values following the value at that index are automatically shifted one position to the right |
    |    Remove(value)    |    Delete    | Deletes the specified data |
    |  RemoveAt(index)  |    Delete    | Deletes the value at the specified index<br/> All values following the deleted value are automatically shifted forward by one position |
    |     Clear() |    Delete    | Deletes all stored data |
    |   Contains(value)   |    Query    | Checks if the data exists in the ArrayList |
    | Count |    Query    | Returns the total number of stored data items |
    | _ArrayList[position] |    Access    | Accesses the value at the **"position"** in the ArrayList |

<br/>

##### Example

```c#
static void Main(string[] args)
    {
        ArrayList al = new ArrayList();
        al.Add(1); //다양한 형식으로 값을 추가
        al.Add("Hello");
        al.Add(3.3);
        al.Add(true);

        foreach(var item in al)
            Console.WriteLine(item);

        Console.WriteLine("---------------");
        al.Remove("Hello"); //Remove() 메소드로 아이템을 삭제함

        foreach (var item in al)
            Console.WriteLine(item);
    }
```

**Output**

```
1
Hello
3.3
True
---------------
1
3.3
True
```

#### Hash Table

- A collection of key-value pairs organized based on the hash codes of the keys

- A data structure that stores data as key-value pairs

- A format where the original text and its translation are saved as a pair

- The larger the dataset, the faster the processing speed relative to the dataset size

- With ArrayList, Stack, and Queue, the order of the stored data is important

  ​  **But** a Hashtable allows you to look up a value immediately using its key

- Any data type (int, float, string, bool, etc.) can be used for the key

  However, if there is a duplicate key, an "Argument Exception" error occurs.

  Speed is reduced due to boxing[^1][^2]
 
 [^2]: Example: int i = 5;  object o = (object) i;










- When storing large amounts of data, it allows for faster access to "stored values" than an ArrayList

  

  ##### How It Works

  ![Image Alt 텍스트]({{link}}/assets/img/OOP/14th/HashTable.png )

  

  - When creating a hash table that stores names as keys and phone numbers as values, with a total of 16 entries,

    Calculate the index value using `index = hash_function(key_name) % 16`, and store `array[16] = "phone_number_value_for_key_name"`.

    Once the data has been stored, when searching for data associated with a key, you can find the index value stored in the array by applying the hash function just once.

  <br/>

  ##### **Issues**

  - The value of hash_function(key) / size_of_array may be duplicated

    - Example: When the key to be stored is an integer and the hash_function is key % 10, 

      When `size_of_array` is 10, the keys 1, 11, 21, and 31 have the same index value.

      This is called a collision

    

    ###### Solution

    - There are various implementation methods
 
 - 1. **Separate chaining** -  A method where each index holds a pointer to a linked list that stores the data
 
 - When a hash collision occurs, the data is linked together using a linked list
 
 - Advantages:
      
           Implementation is relatively simple compared to open addressing, as it only requires a linked list
 
 <br/>
 
 2. **Open Addressing** - A method where data is inserted into a different bucket when a hash collision occurs
 
 - Advantages: 
 1. Unlike chaining, it does not require pointers, and no additional memory beyond the allocated space is needed
 2. Low overhead during insertion/deletion
 3. More efficient when there is a small amount of data

  

  <br/>

- Differences from ArrayList

  - ArrayList: Retrieves data using an index
  - HashTable: Retrieves a value using a key

  

  

  | Method |    Property    | Information |
  | :---------------: | :--------: | :----------------------------------------------------------: |
  |    Add(key, value)    | Save (Add) | Adds **key** and **value** to the HashTable |
  |    Remove(key)     |    Delete    | Deletes the corresponding data (**key** and **value**) from the HashTable |
  | Clear() |    Delete    | Deletes all stored data |
  | Count |    Info    | Retrieves the total number of pairs currently stored |
  |  ContainsKey(key)  |    Info    | Checks if the **key** exists in the HashTable<br/> Returns True if the value exists <br/> Returns False if the value does not exist |
  | ContainsValue(key) |    Info    | Checks if the **** exists in the HashTable<br/> Returns True if the value exists <br/> Returns False if the value does not exist |
  | ht[key] |    Access    |    Accesses the **value** stored under the specified **key** in the HashTable     |

  ##### Example

```c#
static void Main(string[] args)
    {
        Hashtable ht = new Hashtable();
        //키, 값 형태로 아이템 추가
        ht["apple"] = "사과";
        ht["banana"] = "바나나";
        ht["orange"] = "오렌지";

        //HashTable에 저장된 키에 해당하는 값 출력
        Console.WriteLine(ht["apple"]);
        Console.WriteLine(ht["banana"]);
        Console.WriteLine(ht["orange"]);
	}
```

**Output**

```
사과
바나나
오렌지
```

#### Queue

##### How It Works

![Image Alt 텍스트]({{link}}/assets/img/OOP/14th/Queue.png )

- Object collection using the FIFO (First In First Out) method

- A structure where the first value entered is the first to be displayed 

- Used in message queues where messages are stored sequentially and processed sequentially

  

  | | Property | Information |
  | :----------: | :----------------------: | :----------------------------------------------------------: |
  | Enqueue(value)  | Store (Add) | Adds data to the end of the queue |
  |  Dequeue()   | Retrieve (Fetch, Delete) |    Retrieves the data at the front and removes the <br/> value from the queue    |
  |   Clear()    | Delete | Deletes all data stored in the Queue |
  |    Count     | Information | Returns the total number of currently stored data items |
  | Contains(value) | Information | Checks if the specified value exists in the Queue. Returns True if the value exists; False if it does not. |
  |    Peek()    | Access | Checks the data at the front of the Queue |

  <br/>

  ##### Example

  ```c#
    static void Main(string[] args)
      {
          //Queue
          Queue queue = new Queue();
          queue.Enqueue(1);
          queue.Enqueue(2);
          queue.Enqueue(3);
  
          foreach (var item in queue)
              Console.WriteLine(item);
    }
  ```

  **Output**

  ```
  1
  2
  3
  ```

  

#### Stack

- An object collection using the LIFO (Last In First Out) method

- Works in the opposite way to a queue

- The data entered first is output last

- An example of stacking boxes and then moving them starting with the top box

  <br/>

  ##### How It Works

  ![Image Alt 텍스트]({{link}}/assets/img/OOP/14th/Stack.png )



|     Property     | Information | Description |
| :----------: | :----------------------: | :----------------------------------------------------------: |
|   Push(value)   | Store (Add) | Adds data to the end of the stack |
|    Pop()     | Remove (Retrieve, Delete) | Retrieves the data at the very end and removes it from the stack |
|   Clear()    | Delete | Deletes all stored information |
|   Count()    | Information | Retrieves the total number of currently stored data items |
| Contains(value) | Information | Checks if the specified value exists in the stack<br/> Returns **true** if the value exists <br/> Returns **false** if the value does not exist |
|    Peek()    | Access | Checks the data at the front of the stack |



##### Example

```C#
static void Main(string[] args)
    {
        //Stack
        Stack stack = new Stack();
        stack.Push(1); //값 추가
        stack.Push(2);
        stack.Push(3);

        while(stack.Count > 0) //스택에서 아이템 제거
            Console.WriteLine(stack.Pop());
}
```

**Output**

```
3
2
1
```
