## 配列

### 一般変数の欠点

- 一度に1つの値しか保存できません

​  -> 一度に2つ以上の値を保存するには、配列が必要である

<br/>

### 特徴

- 同じデータ型の値を複数保存できる
- int、char、float、クラスなど、さまざまなデータ型を使用できる
- 変数名 = 配列名

<br/>

### 宣言方法

1) 初期化と同時に値を宣言する

   ~~~c#
   データ型[] 配列名 = new データ型[サイズ]{値1, ..., 値n}
   ~~~

   

2) 実行前に値を宣言する

~~~c#
データ型[] 配列名 = {値1, ..., 値n}
~~~

<br/>

### アプローチ方法

- **配列名[index]**

- 配列はnew演算子を使用し、new クラス名()のような形式をとる
- 「**.**」演算子を使用して、プロパティやメソッドを呼び出すことができる

- 配列のインデックスは0から始まり、最後の要素は「配列名.length-1」となる

- System.Index、^(キャレット) [^1]演算子を使用すると、配列へのアクセスがより簡単になる

  <br/>

### パラメータ

#### メソッド

##### 定義

- 配列型をパラメータとして指定する

  ~~~c#
  アクセス修飾子 戻り値の型 関数名(配列型[] 配列名) {} //配列型をパラメータとして指定
  ~~~

<br/>

##### 呼び出し

- **メソッド名(配列名);**
- 例

~~~c#
public static void Main(string[] args)
    {
 void Func(int[] arr)
    {
 foreach(int i in arr)
 Console.WriteLine(i);
    }
 
 int[] testArray = {1,2,3};
 Func(testArray);
    }
~~~

**結果**

~~~
1
2
3
~~~

<br/>

##### 戻り値の型

###### 宣言部

- 戻り値として配列を指定する

~~~c#
アクセス修飾子 配列型[] 関数名(引数型 引数名) {}
~~~

###### リターン

- 返される配列の型に合わせて配列を宣言する

  ~~~c#
  配列型[] 配列名 = 関数名();
  ~~~

  <br/>

#### 値呼び出し方式

##### 参照渡し

- 呼び出されたメソッドで配列の値を変更した場合、呼び出したメソッド側でもその変更が反映されるようにすること

- **ref** キーワードを使用して、配列への参照を渡す

  - 住所を別の名前で共有する

- サンプルコード

  ~~~c#
  using System;
 
 class Program
  {
 static void ArrayIdxChange(ref char[] array, int originalIdx, int changeIdx)
 {
 char tempString = array[originalIdx];
 array[originalIdx] = array[changeIdx];
 array[changeIdx] = tempString;
 }
  
      static void Main(string[] args)
 {


char[] testArr = { 'a', 'b', 'c', 'd', 'e' };
 int originalIdx, changeIdx;
 Console.Write("置き換える文字の元の位置は？ >>");
          originalIdx = Convert.ToInt32(Console.ReadLine());
 Console.Write("選択した文字と置き換える文字の位置は？ >>");
 changeIdx = Convert.ToInt32(Console.ReadLine());
 
 Console.Write("元の配列 >>");
          foreach (char c in testArr)
 Console.Write(c);
 Console.WriteLine();
 ArrayIdxChange(ref testArr, originalIdx, changeIdx);
 Console.Write("変更後の配列>>");
 foreach (char c in testArr)
 Console.Write(c);
 }
  }
  ~~~
  
  **結果**
 
 ~~~
  置換する文字の元の位置は？ >>1
  選択した文字と置換する文字の位置は？ >>2
  元の配列>>abcde
  変更後の配列>>acbde
  ~~~

#### 関連するエラー

##### 1. IndexOutOfRangeException

- 配列の無効なインデックスの値にアクセスしようとしたときに発生する

- 例

  ~~~c#
  using System;
 
 class Program
  {
 static void ArrayIdxChange(ref char[] array, int originalIdx, int changeIdx)
 {
 char tempString = array[originalIdx];
 array[originalIdx] = array[changeIdx];
          array[changeIdx] = tempString;
 }
 
 static void Main(string[] args)
 {
 char[] testArr = { 'a', 'b', 'c', 'd', 'e' };
 ArrayIdxChange(ref testArr, 3, 6);
          Console.Write("変更後の配列>>");
 foreach (char c in testArr)
 Console.Write(c);
 }
  }
  ~~~

  **エラー結果**

  ~~~
  System.IndexOutOfRangeException: 「インデックスが配列の範囲外です。」
  ~~~

  <br/>

  ##### 2. NullReferenceException

  - 配列や配列の要素が指定されていない状態で、メソッドやプロパティにアクセスしようとした際に発生
  - サンプルコード

  ~~~c#
  using System;
 
 class Program
  {
 static void ArrayIdxChange(ref char[] array, int originalIdx, int changeIdx)
 {
 char tempString = array[originalIdx];
 array[originalIdx] = array[changeIdx];
 array[changeIdx] = tempString;
 }
  
      static void Main(string[] args)
 {
 char[] testArr = null;
 ArrayIdxChange(ref testArr, 3, 6);
 Console.Write("変更後の配列>>");
 foreach (char c in testArr)
 Console.Write(c);
 }
  }
  ~~~

  **エラー結果**

  ~~~
  System.NullReferenceException: 「オブジェクト参照がオブジェクトのインスタンスに設定されていません。」
  ~~~

  <br/>

  ##### 3. ArgumentException

  - 配列の初期化時、またはメソッドに誤った引数が渡された場合に発生

  <br/>

  ##### 4. ArrayTypeMismatchException

  - 配列の型を誤った方法で変更しようとした際に発生する

### Arrayクラス

#### 定義

- 配列を作成、操作、検索、およびソートし、共通言語ランタイムにおいてすべての配列の基本クラスとしての役割を果たせるようにするメソッドを提供する[^2]

  <br/>

#### 属性

| 名前 | 意味 |
| :------------- | :----------------------------------------------------------- |
| IsFixedSize    | Arrayのサイズが固定されているかどうかを取得します |
| IsReadyOnly    | Arrayが読み取り専用かどうかを取得します |
| IsSynchronized | Arrayへのアクセスが同期化されており、スレッドセーフであるかどうかを取得する |
| Length | Arrayのサイズを取得する |
| LongLength     | Arrayのすべての次元にある要素の総数を表す64ビット整数を取得する |
| MaxLength | 配列に含めることができる最大要素数を取得します |
| Rank | Arrayの次元数を返します <br> 例) 1次元 : 1<br/>2次元: 2     |
| SyncRoot | Arrayへのアクセスを同期するために利用可能なオブジェクトを取得します |

<br/>

| メソッド名 | 意味 |
| ------------------------------------------------------------ | ------------------------------------------------------------ |
| Clear(Array) | 配列の各要素を削除 |
| Clear(Array, Int32, Int32) | 配列の各要素の型が持つデフォルト値で配列のサイズを設定 |
| Clone() | 配列の部分コピーを作成 |
| ConstrainedCopy(Array, Int32, Array, Int32, Int32) | 指定したインデックスから配列の範囲をコピーし、指定された宛先インデックスから始まる別の配列に貼り付ける |
| ConvertAll<TInput, TOutput>(TInput[], Converter<TInput, TOutput>) | 選択した配列を別の型の配列に変換する |
| Copy(Array, Array, Int32) | 配列を最初の要素からコピーし、別の配列の最初の値から貼り付ける | <br/>32ビット整数で長さが指定される |



[^1]: コレクションの最後の要素から逆順にインデックスを指定可能
[^2]: https://learn.microsoft.com/ko-kr/dotnet/api/system.array?view=net-7.0
