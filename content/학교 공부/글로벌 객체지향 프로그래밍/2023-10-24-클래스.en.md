## Class

### Definition

- A class is a definition of the objects that the program operates on, using the `class` keyword
- Using a class and the `new` keyword, you can create objects (instances)

<br/>

### How to Declare

~~~C#
access_qualifier class ClassName {
  // Implementation of fields, properties, methods, and events
    access_qualifier data_type variable_name;
    
    access_modifier data_type variable_name {
 get{}
 set{}
    }
 
 access_modifier return_type function_name(parameter1, ..., parameterN) {
 ...
    }
 
 access_modifier event EventHandler event_variable;
}
~~~

<br/>

### Components

- Create and use objects from a class

| Type | Description |
| -------------- | ------------------------------------------------------------ |
| Method | A block of code that runs within a class<br/>Names are often defined using a verb or verb+noun format<br> There are no restrictions on the number of parameters, but the number of return values must match the number of parameters<br><br/> **access_modifier data_type method_name(parameters) { return return_value; }**<br> Can be created using **MethodName(parameters)** |
| Property | A member that allows data within a class to be accessed and modified from outside<br/> In object-oriented languages such as C#, it is used for the purpose of encapsulation<br/> A property requires one field and two methods: get and set |
| Field    | Used to maintain the state of a class object<br/> Even within the same class, objects have different states depending on the fields they possess<br/> Depending on the access modifier, they can be exposed to external objects or derived classes |
| Event | Used to communicate a specific internal state of an object or the occurrence of an event to the outside world<br/> Example: When a button is clicked, notifying all external objects subscribed to the button event that the button has been activated |

<br/>

### Nested Classes

#### Definition

- Defining one class within another
- Allows for grouping classes and restricting their scope

#### How to Use

~~~c#
class Class1{
    class nestedClassName{
 ...
    }
}
~~~

**Example**

~~~C#
class Class1{
  int outerValue;
    class Class2{
 Class1 class = new Class1();
 class.outerValue = 1;
    }
}
~~~

<br/>

### The partial class

#### Definition

- A method of implementing a class by splitting its contents into separate files
- Although the files are separate, they are combined into a single class during the compilation process
- Classes can be split using the `partial` keyword

#### Important Notes

- All split files must use the same access modifier
  - Otherwise, a conflict will occur
- Since C# does not support multiple inheritance, a compilation error will occur if the parent classes are different

<br/>

### Object

#### Definition

- Creating an instance of a class in heap memory using the `new` keyword

#### How to Use

~~~c#
ClassName ObjectName = new ClassName();
~~~

##### Example

~~~c#
Circle circle = new Circle();
~~~

<br/>

#### How to Call

- If there is no "Static" keyword

  - **ObjectName.Member;**
  - Example)

  ~~~c#
  dog.name;
  ~~~

- If the `static` keyword is present

  - **ClassName.Member;**
  - Example)

  ~~~c#
  Dog.AnimalName
  ~~~


<br/>

### This

- Used to access a member of the object itself
- From outside the object, use **objectName.method** or **objectName.variable** to access the object's methods or variables
- Used within the object to access its own variables or methods

<br/>

### Constructor

- A method used for initialization, called when an instance is created
- It shares the same name as the class and can accept any number of arguments
- It has no return type

<br/>

## Access Modifiers

### Definition

- Used to restrict access to members within a class

### Types

#### private

- As the default access modifier, it can only be used within the class

#### public

- Accessible from outside the class

#### protected

- Accessible only from subclasses that inherit from it

#### internal

- Allows access to assemblies (e.g., .NET-based application deployment, version control, reuse, etc.)
- When creating a project, variables and types declared as `internal` are accessible only within the same project

#### protected internal

- Access is restricted to child classes that inherit from the same class within the same project

<br/>

### Usage Guidelines

- Declare fields as private to prevent external access
  - Adhere to **encapsulation**, one of the principles of object-oriented programming
- Declare methods as public
- This is known as information hiding
