## 相続

- 親クラスの属性やメソッドを継承して新しいクラスを作成すること <br>

- JavaやC#では多重継承をサポートしていない（インターフェースの場合は多重継承をサポートしている）

  - C++では多重継承をサポートしている



## 相続のメリット

* 重複するコードを各クラスごとに記述する必要がなくなる <br>

* クラスの変更や拡張が容易

<br>

## 使い方

```c#
class 부모클래스{
    //내용
}
class 자식클래스 : 부모클래스{
    //부모 클래스의 모든 상태, 내용이 전달됨
    자식 클래스만의 메소드
    자식 클래스만의 데이터
}
```
親クラスのprivateプロパティとコンストラクタは生成されない



## 使用例

RPGゲームで銃、剣、弓を使用する場合に必要な事項

| 銃 | 剣 | 弓 |
| :----------------: | :----------: | :------------------: |
| 関数：攻撃、リロード | 関数：攻撃  |     関数：攻撃 |
| 変数：攻撃力、弾倉 | 関数：攻撃力 | 関数：攻撃力、射程 |

<br/>

​  3つの武器すべてにおいて、「攻撃」という関数と「攻撃力」という変数が重複している

​  しかし、「リロード」という関数や、「弾倉」と「射程」という変数は、すべての武器で使用されるわけではない

​  複数の武器を追加する場合は、同じ関数や変数を作成する必要がある



**解決策：** 共通の関数と変数を親クラスに定義し、子クラスで継承して実装する

```c#
class Weapon{
	public void Attack() {}
    protected int power;
}

class Gun : Weapon{
    public void Reload() {}
    private int magazine;
}
class Knife : Weapon { //내용 }
class Bow : Weapon { private int reach; }
```



## 基本

* メソッドでの使用

  <br/>

  ### 使い方

  ```c#
  base.부모메소드();
  ```

  <br/>

* コンストラクタ内で呼び出され、親クラスのコンストラクタを呼び出した後、自身のコンストラクタを呼び出す

* Child コンストラクタ：**base()**



**例 **

```c#
class Base{
	public void BaseMethod() {}
}

class Derived : Base{
    public void DerivedMethod(){ base.BaseMethod(); }
}
```



## 密封

* 「封印される」という意味で、改ざんから保護され、外部からの変更や修正が不可能な状態 <br/>



* **Class**の前にSealedがある場合
  * クラスの継承が不可能
* **メソッド**の前にSealedがある場合
  * メソッドの継承（オーバーライド）が不可能



​  **サンプルコード**

```c#
sealed class Parent{
	public int num;
    public Parent() { Console.WriteLine("부모 클래스의 생성자가 호출 되었습니다."); }
}

class Child : Parent{
    public int num;
    public Child(int num) {
        this.num = num;
        Console.WriteLine("자식 클래스의 생성자가 호출되었습니다.");
    }
    public void DisplayValue(){ Console.WriteLine("num의 값은 {0}입니다.", num); }
}
```
