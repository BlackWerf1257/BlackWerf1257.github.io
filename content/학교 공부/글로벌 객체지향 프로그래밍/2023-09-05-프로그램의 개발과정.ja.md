## プログラムの開発過程

### 開発者 vs コンピュータ

- 開発者：数字、アルファベット、特殊文字を理解できる
  - コンピュータ：2進数のみを読み取ることができる

<br/>

### プログラミング言語

#### 概念

- 人間がコンピュータに命令を出すために、人が理解できる英語の単語を使って記述できる言語
- 規則がある

<br/>

### 機械語

- コンピュータが使用する言語であり、CPUはこの機械語を解釈して処理を行う

<br/>

### ビルド

- 実行ファイルを生成する作業
- ビルドが完了すると、機械語に新たに翻訳されたファイルが生成される
  - オブジェクトファイルとも呼ばれる
- リンクとは、複数のオブジェクトファイルを結合して1つのプログラムを作成する作業
- C#では、プログラムが共通して利用できる機能をあらかじめ用意しており、これをライブラリと呼ぶ
- 実行ファイルを作成するには、この機能を提供するオブジェクトファイルをリンクする必要がある

<br/>

### コンパイルエラーとリンクエラー

#### コンパイルエラー

- ソースを機械語に変換する過程で発生するエラーであり、文法構造に関するエラーである

#### リンクエラー

- ソースコードとライブラリをリンクする過程で発生するエラー
- ソースコードの定義が適切か、呼び出しが正しく行われているかを確認する必要がある

#### 実行ファイルの作成後の論理エラー

- 期待した結果が得られなかった場合
  - デバッグを行うことで解決可能です

<br/>

### 予約語

- プログラミング言語において、特定の目的のためにあらかじめ定義されているキーワードであり、他の目的では使用できない
- 名前空間名、クラス名、メソッド名、変数名は開発者が定義できる

<br/>

### 名前空間

#### 定義

- 共同作業中は、クラス名やメソッド名、変数名などが重複する可能性がある
- こうした衝突を防ぐために、自分のコードをまとめておくこと

#### 使い方

~~~c#
namespace 名前空間名{
class クラス名{
  ...
    }
}
~~~

<br/>

- 他のクラスから当該クラスを使用する場合は、**using 名前空間名** を使用して利用可能である

<br/>

### クラス

#### 定義

- プログラミングの対象となる事象を「class」というキーワードを用いて統合したもの
- クラス内部には、動作を表すメソッドや属性、メンバー変数を含むフィールドがある

#### 使い方

~~~c#
アクセス修飾子 class クラス名{
  戻り値 メソッド名(引数){
    ...
  }
 
 データ型 変数名;
}
~~~
