## クラス

### 定義

- プログラムの対象となる事物を「class」というキーワードを用いて定義したもの
- クラスを用いて「new」キーワードを使用することで、オブジェクト（インスタンス）を作成することができる

<br/>

### 宣言方法

~~~C#
アクセス修飾子 class クラス名{
  //フィールド、プロパティ、メソッド、イベントの実装
    アクセス修飾子 データ型 変数名;
    
    アクセス修飾子 データ型 変数名{
 get{}
 set{}
    }
 
 アクセス修飾子 戻り値型 関数名(パラメータ1, ..., パラメータn){
 ...
    }
 
 アクセス修飾子 event イベントハンドラ イベント 変数;
}
~~~

<br/>

### 構成要素

- クラスからオブジェクトを作成して使用する

| 種類 | 説明 |
| -------------- | ------------------------------------------------------------ |
| メソッド (Method) | クラス内で動作するコードブロック<br/>動詞または動詞＋名詞の形式で名前が付けられることが多い<br> パラメータの制限はないが、値はパラメータの数と同じでなければならない<br><br/> **アクセス修飾子 データ型 メソッド名(パラメータ) { return 戻り値; }**を使用して作成可能<br> **メソッド名(パラメータ)**を使用してアクセス可能 |
| プロパティ(Property) | クラス内のデータを外部から使用および変更できるように設定できるメンバー<br/> C#のようなオブジェクト指向言語では、カプセル化の目的で使用される<br/> プロパティは、1つのフィールドに対してgetとsetという2つのメソッドを必要とする |
| フィールド(Field)    | クラスオブジェクトの状態を維持するために使用される<br/> 同じクラスであっても、オブジェクトはそれぞれのフィールドを持つため、互いに異なる状態を持つことになる<br/> アクセス修飾子の設定により、外部オブジェクトや派生オブジェクトからも公開可能である |
| イベント(Event)  | オブジェクト内部の特定の状態や、何らかの事象が発生したことを外部に伝達する目的で使用される<br/> 例：ボタンをクリックした場合、そのボタンイベントに登録されているすべての外部オブジェクトに、ボタンが動作したことを通知すること |

<br/>

### ネストされたクラス

#### 定義

- 1つのクラス内で別のクラスを定義すること
- クラスをグループ化したり、クラスの使用範囲を制限したりすることが可能

#### 使い方

~~~c#
class Class1{
    class ネストされたクラス名{
 ...
    }
}
~~~

**例**

~~~C#
class Class1{
  int outerValue;
    class Class2{
 Class1 class = new Class1();
 class.outerValue = 1;
    }
}
~~~

<br/>

### partialクラス

#### 定義

- クラスの内容を別のファイルに分割して実装したもの
- ファイルは別々だが、コンパイル時に1つのクラスとして結合される
- partialキーワードを使用してクラスを分割することができる

#### 注意点

- 分割されたファイルはすべて同じアクセス修飾子を使用する必要があります
  - 異なる場合、競合が発生します
- C#は多重継承をサポートしていないため、親クラスが異なる場合、コンパイルエラーが発生します

<br/>

### オブジェクト

#### 定義

- 作成されたクラスを「new」キーワードを使用して、ヒープメモリ上に生成すること

#### 使い方

~~~c#
クラス名 オブジェクト名 = new クラス名();
~~~

##### 例

~~~c#
Circle circle = new Circle();
~~~

<br/>

#### 呼び出し方法

- Static 予約語がない場合

  - **オブジェクト名.メンバー;**
  - 例）

  ~~~c#
  dog.name;
  ~~~

- Static 予約語がある場合

  - **クラス名.メンバー;**
  - 例）

  ~~~c#
  Dog.AnimalName
  ~~~


<br/>

### これ

- 自身のオブジェクトのメンバにアクセスする際に使用する
- オブジェクトの外部からオブジェクトのメソッドや変数にアクセスする際は、**オブジェクト名.メソッド** または **オブジェクト名.変数** を使用してアクセスする
- オブジェクトの内部から自身の変数やメソッドにアクセスする際に使用する

<br/>

### コンストラクタ

- 初期化のためのメソッドであり、インスタンス生成時に呼び出される
- クラス名と同じ名前を使用し、引数として任意の値を受け取ることができる
- 戻り値がない

<br/>

## アクセス制限子

### 定義

- クラス内部のメンバへのアクセスを制限する役割を果たす

### 種類

#### プライベート

- デフォルトのアクセス制限子であり、クラス内部でのみ使用可能

#### 公開

- クラスの外部からアクセス可能

#### 保護済み

- 継承されたサブクラスからのみアクセス可能

#### 内部

- アセンブリ（.NETベースのアプリケーションの配布、バージョン管理、再利用など）へのアクセスを許可する
- プロジェクト作成時、internalとして宣言された変数および型は、同一プロジェクト内でのみアクセス可能である

#### 保護された内部

- 同一プロジェクト内で、継承を受けた子クラスからのみアクセス可能

<br/>

### 使用上の原則

- フィールドをprivateに指定し、外部からのアクセスを遮断する
  - オブジェクト指向の原則の一つである**カプセル化**を遵守する
- メソッドはpublicに指定する
- これを情報の隠蔽と呼ぶ
