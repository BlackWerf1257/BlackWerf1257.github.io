## 静的

- ![画像の代替テキスト]({{link}}/assets/img/OOP/10th/Memory_Structure.png )
- 使用場所に応じて、**静的クラス**、**静的メソッド**、**静的メンバー関数**に分類される

- ヒープ領域とは、new キーワードを用いてメモリ上に動的に割り当てられる領域を指す

- Stack領域に格納される変数には、ローカル変数と引数が含まれる
- BSS領域は、初期化されていないグローバル変数と静的変数が存在する領域であり、すべての値は0に初期化される
- Staticはdata領域に相当し、プログラムの開始時に読み込まれ、プログラム終了時にメモリが解放される。
- プログラム内には1つしか存在しない

<br/>

- クラスが管理する変数であり、static ではないクラス変数やメンバ変数はオブジェクトが管理する
- 宣言された位置に応じて、下位の構成要素はすべて static でなければならない。

- スタックとヒープは使用後、ガベージコレクタによって処理される
 - BSSは該当しない

<br>

### メモリ領域

#### データ領域

- 静的変数が割り当てられる領域
- プログラムの開始と同時に割り当てられ、プログラムの終了時にメモリから削除される

#### スタック領域

- 関数呼び出し時にローカル変数と引数が格納される領域
- 関数呼び出し後に破棄される
- 最後に呼び出された変数のメモリから解放される

#### ヒップエリア

- プログラマーが管理するメモリ領域
- 動的割り当てによって生成されるメモリ領域である
- 先に割り当てられたメモリから解放される





<br/>

### 静的クラス

- `new` キーワードを使用してインスタンスを作成できない
- すべてのメンバは static として宣言しなければならない
- コンストラクタを含めることはできない（static では `new` を使用しない）

- オブジェクトの初回呼び出し時に生成され、プログラム終了時に破棄される
  - 静的クラスはどこからでもアクセス可能
- グローバルにアクセスするユーティリティクラスを作成する際に有用

<br/>

**例**



```c#
public static class Calculator
{
    public static int Plus(int a, int b) { return a + b; }
    public static int Minus(int a, int b) { return a - b; }
}

public class Program
{
    public static void Main(string[] args)
    {
        int addResult = Calculator.Plus(5, 3);
        int minusResult = Calculator.Minus(5, 3);

        Console.WriteLine(addResult);
        Console.WriteLine(minusResult);
    }
}
```

```
결과 :
5
2
```

### 静的メソッド

- インスタンスを直接作成せずに呼び出しが可能

- メソッド内部ではオブジェクトのメンバを参照できない

- インスタンスからは呼び出せません

  - **オブジェクト名.メソッド名()** は使用不可

  <br/>

### 静的フィールド（メソッド）

- インスタンスを直接生成せずにアクセス可能
- データ型の前にstaticキーワードを宣言して生成
- どこからでもアクセス可能
  - 主にグローバルにアクセスする必要がある変数に使用
- スタティックメモリ領域に保存 → プログラム終了時までメモリ上に存在
- いくつ生成しても、クラスごとに1つの値しか保持しない
- 使用理由：インスタンス間で値を共有するため



```c#
public class Book
{
    public static int count = 0;
    private string name;
    public Book(string name) {
        this.name = name;
        count++;
    }
}


public class Program
{
    public static void Main(string[] args)
    {
        //정적 필드
        Book book1 = new Book("Harry Potter");
        Book book2 = new Book("The Lord of the Rings");
        Console.WriteLine("Created Book Count : " + Book.count);
    }
}
```

```
출력 :
Created Book Count : 2
```



- xxxxxxxxxx アクセス修飾子 class クラス名{    戻り値 メソッド名(パラメータ){ ...    } データ型 変数名;}c#
  - したがって、インスタンスを複数生成しても、クラス内には1つしか存在しないため、book1で+1、book2で+1を行い、値は2となる
  - もしcountにstaticが付いていない場合、各インスタンスごとにcountが1つずつ生成されるため、値は0になる
