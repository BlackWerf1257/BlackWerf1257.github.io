## Interface

### Definition

- Contains methods, properties, events, and indexers, but does not define them directly; it only holds their definitions
- An interface defines only the form of methods; the implementation of those methods is handled by the classes that inherit from it



### How to Declare

```c#
interface 인터페이스명{
  메소드();
  속성;
  이벤트;
}
```

- Methods and properties of an interface are public by default, even if they do not have a public access modifier.
- When naming an interface, the letter **'I'**, which stands for "Interface," is implicitly prefixed to the interface name.



### Classes that inherit from an interface

- Classes that inherit from an interface must fully implement all methods defined in that interface.
- Methods declared in an interface must be implemented as **public**.



### Example

```c#
interface IAnimal{
    void Sound(); //세부 내용은 파생클래스에서 구현
}

class Pig : IAnimal{ //IAnimal 인터페이스를 상속
    public void Sound(){
        Console.WriteLine("Pig Sound : Wee Wee");
    }
}

class Cat : IAnimal{
    public void Sound(){
        Console.WriteLine("The Cat says : wee wee");
    }
}


class Program{
    static void Main(string[] args){
        IAnimal animal1 = new Pig(); //IAnimal 인터페이스를 상속받은 Pig 클래스를 선언함
        IAnimal animal2 = new Cat(); //IAnimal 인터페이스를 상속받은 Cat 클래스를 선언함
        
        animal1.Sound();
        animal2.Sound();
    }
}
```

```
Output : 

Pig Sound : Wee Wee
The Cat says : wee wee
```



- The classes `animal1` and `animal2` both inherit from the `IAnimal` interface, but they output different results.
- Although the method call syntax is the same, the code executed is completely different depending on the type of object created.
