## インターフェース

### 定義

- メソッド、プロパティ、イベント、インデクサーなどを備えているが、それらを直接定義するのではなく、定義のみを保持する
- インターフェースではメソッドの型のみを定義し、メソッドの実装は継承したクラスで行われる



### 宣言方法

```c#
interface 인터페이스명{
  메소드();
  속성;
  이벤트;
}
```

- インターフェースのメソッドとプロパティは、publicというアクセス修飾子がなくても、デフォルトでpublicとなる
- インターフェースに名前を付ける際は、暗黙的にインターフェース名の前に「Interface」を意味する**'I'**を付ける



### インターフェースを継承したクラス

- インターフェースを継承したクラスでは、インターフェース内のメソッドをすべて実装しなければならない
- インターフェースに宣言されたメソッドは、必ず**public**として実装しなければならない



### 例

```c#
interface IAnimal{
    void Sound(); //세부 내용은 파생클래스에서 구현
}

class Pig : IAnimal{ //IAnimal 인터페이스를 상속
    public void Sound(){
        Console.WriteLine("Pig Sound : Wee Wee");
    }
}

class Cat : IAnimal{
    public void Sound(){
        Console.WriteLine("The Cat says : wee wee");
    }
}


class Program{
    static void Main(string[] args){
        IAnimal animal1 = new Pig(); //IAnimal 인터페이스를 상속받은 Pig 클래스를 선언함
        IAnimal animal2 = new Cat(); //IAnimal 인터페이스를 상속받은 Cat 클래스를 선언함
        
        animal1.Sound();
        animal2.Sound();
    }
}
```

```
Output : 

Pig Sound : Wee Wee
The Cat says : wee wee
```



- クラスanimal1とanimal2は同じIAnimalインターフェースを継承しているが、出力される内容は異なる。
- メソッドの呼び出し形式は同じだが、生成されたオブジェクトの型によって、実行される内容は全く異なる。
