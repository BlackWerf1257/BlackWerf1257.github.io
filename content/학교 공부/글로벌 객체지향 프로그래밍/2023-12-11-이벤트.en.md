## Delegate

#### Definition

- A technique for calling a method on behalf of another
- Use the `delegate` keyword for declaration

### How to Use

#### How to Declare

```c#
접근제한자 delegate 반환타입 대리자 이름(매개변수 목록);
```

**Example**

```c#
public delegate void MyTestDelegate(string value);
```

**How to Call**

```c#
//메소드가 여러번 사용되는 경우
대리자이름(매개변수 값);

//메소드가 한번만 사용되는 경우
대리자이름 객체이름 = delegate 반환타입(매개변수);
```

**Examples of Use**

```c#
public static void MyTestVoid(string value)
{
  Console.WriteLine(value);
};

MyTestDelegate delegate1 = MyTestVoid; // MyTestVoid 메소드를 del1에 할당
delegate1("Method A Called");
MyTestDelegate delegate2 = delegate void (string value); //한번만 사용되는 메소드에 델리게이트 하는 경우

```



<br/>

#### Important Notes

- Create a method to assign to the delegate - The return type and method must match the delegate declaration.



#### Why Use It

- Useful when you need to pass a method as a parameter to another method or combine methods

  - Without using delegates, you would have to write numerous methods or logic

    -> Increased code length



## Action

### Concept

- A delegate that can point to a method with no inputs (parameters) or outputs (return types)

### How to Use

#### Declaration

```c#
Action 변수;
```

#### Allocation

```c#
변수 += 메소드 이름;
```

#### Call

```c#
변수();
```



### Example

```c#
public class ActionTest : MonoBehaviour
{
    private Action action;
    
    // Start is called before the first frame update
    void Start()
    {
        action += Alpha;
        action += Beta;

        action(); //등록된 모든 메소드가 일괄 실행
    }
    void Alpha() => Debug.Log("Alpha");
    void Beta() => Debug.Log("Beta");
}
```



## Event

### Concept

- An event that triggers a chain reaction

- Although no action is taken, when an event occurs, the registered handlers are executed in sequence

- The event doesn't care what actions the event listener performs

- An event executes methods without knowing the internal implementation of the methods registered to it

  - Resolving the Tight Coupling[^1] Issue

    - Loose coupling[^2]

      [^2]: When handled as an event, the coupling between the two pieces of code is weak, making it easy to modify the code.

      

    [^1]: This means that the two codes are tightly coupled, making it difficult to modify them.

<br/>

- Event listeners
  - Methods that are executed when an event occurs
  - Registering for an event is often described as the event listener "subscribing" to the event

<br/>

## Event

- Variables of the delegate type can be used with the `event` keyword.
- If a delegate variable is declared as an `event`, the delegate cannot be executed outside the class.
  - It is executed only within the class.
- Using this approach prevents events from being triggered by code that does not use the `event` keyword.
