## 配列

### 概念

- 同じデータ型を格納できる
- インデックスを介してデータにアクセスし、データの挿入や値の変更が可能である

<br/>

## コレクション

### 概念

- さまざまなデータを保存できる
- **System.collections** ネームスペースを宣言することで使用可能

<br/>

### 種類

#### ArrayList

- 必要に応じてサイズが動的に拡大するオブジェクト配列
- 配列に似たコレクションであり、インデックスを使用して値にアクセスでき、直接読み書きが可能
- あらゆる型の値を格納可能
- 値を取得する際、var型を使用して暗黙的に型変換が可能
- 追加・削除・変更・検索が可能

<br/>

- ##### 配列との違い

  - 配列は作成時にサイズを指定する

  - ArrayListは作成後、値の追加や削除に応じてサイズが動的に変化する

    

    

    | メソッド |    属性    | 説明 |
    | :--------------: | :--------: | :----------------------------------------------------------: |
    |     Add(値) | 保存(追加) | ArrayListの保存された情報の末尾に値を追加する |
    | Insert(位置, 値) | 保存(追加) | 指定したインデックスの位置に値を追加する<br/>その位置にある値以降、後ろにある値はすべて1つずつ自動的に後ろへ移動する |
    |    Remove(値)    |    削除    | 該当するデータを削除します |
    |  RemoveAt(位置)  |    削除    | 指定したインデックスの位置にある値を削除します<br/> 削除された値の後ろにある値から、後ろにある値はすべて1つずつ自動的に前に移動します |
    |     Clear() |    削除    | 保存されているすべての情報を削除します |
    |   Contains(値)   |    検索    | ArrayList内にデータがあるか検索します |
    | Count |    検索    | 保存されたデータの総数を返します |
    | _ArrayList[位置] |    アクセス    | 該当するArrayListの**"位置"**にある値にアクセスします |

<br/>

##### 例

```c#
static void Main(string[] args)
    {
        ArrayList al = new ArrayList();
        al.Add(1); //다양한 형식으로 값을 추가
        al.Add("Hello");
        al.Add(3.3);
        al.Add(true);

        foreach(var item in al)
            Console.WriteLine(item);

        Console.WriteLine("---------------");
        al.Remove("Hello"); //Remove() 메소드로 아이템을 삭제함

        foreach (var item in al)
            Console.WriteLine(item);
    }
```

**出力**

```
1
Hello
3.3
True
---------------
1
3.3
True
```

#### ハッシュテーブル

- キーのハッシュ値に基づいて構成されたキーと値のペアの集合

- データをキーと値として保存する構造

- 原文と翻訳文がペアで保存される形式

- データのサイズが大きくなるほど、処理速度は相対的に速くなる

- ArrayList、Stack、Queueは、保存されたデータの順序が重要である

  ​  **しかし**、ハッシュテーブルならキーを使って一度に値を検索できる

- 型はどのようなタイプ（int、float、string、bool……）でも使用可能です

  ​  しかし、同じキーが存在する場合、「Argument Exception」エラーが発生する

  ボクシング[^1][^2]の問題により速度が低下する
 
 [^2]: 例：int i=5;   object o  = (object) i;










- 大量のデータを保存する場合、ArrayListよりも高速に「保存された値」にアクセスできる

  

  ##### 動作方式

  ![画像の代替テキスト]({{link}}/assets/img/OOP/14th/HashTable.png )

  

  - データ全体の数が16個である場合、名前をキー、電話番号を値として保存するハッシュテーブルを作成する際、

    index = hash_function(キー名) % 16 によってインデックス値を算出し、array[16] = "キー名の電話番号" を格納する

    データを保存した後、キーに対応するデータを検索する際、ハッシュ関数を一度実行するだけで、配列内に保存されたインデックス値を見つけることができる

  <br/>

  ##### **問題点**

  - hash_function(key) / size_of_array の値が重複する可能性がある

    - 例）保存したいキーが整数型で、ハッシュ関数が key % 10 の場合、

      ​  size_of_arrayが10の場合、キー1、11、21、31は同じインデックス値を持つことになる

      これを「衝突」という

    

    ###### 解決方法

    - 様々な実装方法が存在する
 
 - 1. **セパレート・チェイニング方式** -  各インデックスに、データを格納するリンクリストへのポインタを保持する方式
 
 - ハッシュ衝突が発生した場合、リンクリストでデータを連結する
 
 - 利点：
      
           連結リストのみを使用すればよいため、実装がオープンハッシュ法に比べて比較的簡単である
 
 <br/>
 
 2. **オープンアドレス法** - ハッシュ衝突が発生した際、別のバケットにデータを挿入する方式
 
 - 利点: 
 1. チェイニングのようにポインタが不要であり、指定されたメモリや追加の保存領域を必要としない
 2. 挿入・削除時のオーバーヘッドが小さい
 3. 指定するデータが少ない場合、より効率的である

  

  <br/>

- ArrayListとの違い

  - ArrayList：データを検索する際、インデックスを使って値を検索する
  - HashTable：キーを使って値を検索する

  

  

  | メソッド |    属性    | 情報 |
  | :---------------: | :--------: | :----------------------------------------------------------: |
  |    Add(キー, 値)    | 保存(追加) | Hashtableに**キー**と**値**を追加 |
  |    Remove(キー)     |    削除    | Hashtableから該当データ(**キー**と**値**)を削除 |
  | Clear() |    削除    | 保存されているすべてのデータを削除 |
  | Count |    情報    | 現在保存されているデータのペアの総数を取得 |
  |  ContainsKey(キー)  |    情報    | HashTableに当該**キー**があるか照会する<br/> 値がある場合 True <br/> 値がない場合 False |
  | ContainsValue(キー) |    情報    | HashTableに当該****があるか照会する<br/> 値がある場合 True <br/> 値がない場合 False |
  | ht[キー] |    アクセス    |    該当するHashTableの**キー**で保存されている**値**にアクセス     |

  ##### 例

```c#
static void Main(string[] args)
    {
        Hashtable ht = new Hashtable();
        //키, 값 형태로 아이템 추가
        ht["apple"] = "사과";
        ht["banana"] = "바나나";
        ht["orange"] = "오렌지";

        //HashTable에 저장된 키에 해당하는 값 출력
        Console.WriteLine(ht["apple"]);
        Console.WriteLine(ht["banana"]);
        Console.WriteLine(ht["orange"]);
	}
```

**出力**

```
사과
바나나
오렌지
```

#### キュー

##### 動作方式

![画像の代替テキスト]({{link}}/assets/img/OOP/14th/Queue.png )

- FIFO（先入れ先出し）方式のオブジェクトコレクション

- 入力された順に出力される仕組み

- メッセージを順次保存し、メッセージキューで順次処理するために使用

  

  | | 属性 | 情報 |
  | :----------: | :----------------------: | :----------------------------------------------------------: |
  | Enqueue(値)  | 保存(追加) | Queueの末尾にデータを追加する |
  |  Dequeue()   | 取り出し(取得、削除) |    先頭のデータを取り出し、<br/> 値をQueueから削除する    |
  |   Clear()    | 削除 | Queueに保存されているすべての情報を削除 |
  |    Count     | 情報 | 現在保存されているデータの総数を返す |
  | Contains(値) | 情報 | Queue内にその値があるか照会する<br/> 値が存在すればTrue <br/> 値が存在しなければFalse |
  |    Peek()    | アクセス | 先頭にあるデータを確認する |

  <br/>

  ##### 例

  ```c#
    static void Main(string[] args)
      {
          //Queue
          Queue queue = new Queue();
          queue.Enqueue(1);
          queue.Enqueue(2);
          queue.Enqueue(3);
  
          foreach (var item in queue)
              Console.WriteLine(item);
    }
  ```

  **出力**

  ```
  1
  2
  3
  ```

  

#### スタック

- LIFO（後入れ先出し）方式のオブジェクトコレクション

- Queueとは逆の仕組みで動作する

- 最初に入力されたデータが最後に出力される

- 箱を積み重ねた後、一番上の箱から移動させる場合が例です

  <br/>

  ##### 動作方式

  ![画像の代替テキスト]({{link}}/assets/img/OOP/14th/Stack.png )



|     属性     | 情報 | 内容 |
| :----------: | :----------------------: | :----------------------------------------------------------: |
|   Push(値)   | 保存(追加) | Stackの末尾にデータを追加 |
|    Pop()     | 取り出し(読み出し、削除) | 最後尾にあるデータを取り出し、Stackから削除 |
|   Clear()    | 削除 | 保存されているすべての情報を削除 |
|   Count()    | 情報 | 現在保存されているデータの総数を取得する |
| Contains(値) | 情報 | スタック内にその値があるかどうかを調べる<br/> 値が存在する場合 **true** <br/> 値がない場合 **false** |
|    Peek()    | アクセス | 一番先頭にあるデータを確認する |



##### 例

```C#
static void Main(string[] args)
    {
        //Stack
        Stack stack = new Stack();
        stack.Push(1); //값 추가
        stack.Push(2);
        stack.Push(3);

        while(stack.Count > 0) //스택에서 아이템 제거
            Console.WriteLine(stack.Pop());
}
```

**出力**

```
3
2
1
```

[^1]: 値型オブジェクトを参照型に変換する操作 ↩
[^2]: 例：int i=5;   object o  = (object) i;
