## Debug Class

- When developing in C#, use Console.WriteLine() or Console.Write() to output data.
  - However, since this debugging functionality continues to run even after deployment, it consumes memory.



- Issues caused by excessive debugging

  - You must delete all of these commands before deployment.

  - There is a risk of accidentally deleting other code while deleting the target code.

    →This can be resolved using conditional compilation statements



**Example**

```c#
#define RELEASE
    class LOG{
        #if RELEASE
            public statoc void print(string str)
	        { Debug.Log(str); }
        #else
            public static void print(string str) { 				//Debug.log(); 
        	}
        #endif
    }
```



- When using the debug class as shown in the example, 

  Debug mode runs only when **#define RELEASE** is used
