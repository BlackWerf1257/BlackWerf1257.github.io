## 委任

#### 定義

- メソッドの代わりに呼び出す手法
- 宣言にはdelegateキーワードを使用する

### 使い方

#### 宣言方法

```c#
접근제한자 delegate 반환타입 대리자 이름(매개변수 목록);
```

**例**

```c#
public delegate void MyTestDelegate(string value);
```

**呼び出し方法**

```c#
//메소드가 여러번 사용되는 경우
대리자이름(매개변수 값);

//메소드가 한번만 사용되는 경우
대리자이름 객체이름 = delegate 반환타입(매개변수);
```

**使用例**

```c#
public static void MyTestVoid(string value)
{
  Console.WriteLine(value);
};

MyTestDelegate delegate1 = MyTestVoid; // MyTestVoid 메소드를 del1에 할당
delegate1("Method A Called");
MyTestDelegate delegate2 = delegate void (string value); //한번만 사용되는 메소드에 델리게이트 하는 경우

```



<br/>

#### 注意事項

- デリゲートに割り当てるメソッドを作成する - 戻り値の型とメソッドは、デリゲートの宣言と一致している必要がある。



#### 利用理由

- メソッドの引数としてメソッドを渡したり、メソッドを組み合わせたりする必要がある場合に便利

  - デリゲートを使用しない場合、数多くのメソッドやロジックを実装する必要がある

    -> コードの長さが増加



## アクション

### 概念

- 入力（引数）と出力（戻り値）を持たないメソッドを指すことができるデリゲート

### 使い方

#### 宣言

```c#
Action 변수;
```

#### 割り当て

```c#
변수 += 메소드 이름;
```

#### 呼び出し

```c#
변수();
```



### 例

```c#
public class ActionTest : MonoBehaviour
{
    private Action action;
    
    // Start is called before the first frame update
    void Start()
    {
        action += Alpha;
        action += Beta;

        action(); //등록된 모든 메소드가 일괄 실행
    }
    void Alpha() => Debug.Log("Alpha");
    void Beta() => Debug.Log("Beta");
}
```



## イベント

### 概念

- 連鎖反応を引き起こす出来事

- 何らかの処理を実行するわけではないが、イベントが発生すると、登録された処理が連鎖的に実行される

- イベントは、イベントリスナーがどのような処理を実行するかには関係ありません

- イベントは、自身に登録されたメソッドの内部実装を知らない状態で、そのメソッドを実行する

  - 堅牢なカップリング[^1]の問題の解決

    - 緩やかな結合[^2]

      [^2]: イベントとして処理する場合、2つのコード間の関連性が弱いため、コードの修正が容易である

      

    [^1]: 2つのコード間の関連性が強いため、コードの修正が困難であることを意味する

<br/>

- イベントリスナー
  - イベントが発生した際に実行されるメソッド
  - イベントへの登録を、イベントリスナーがイベントを購読すると言う

<br/>

## イベント

- デリゲート型の変数は、`event` キーワードを付けて使用可能
- デリゲート変数を `event` として宣言すると、クラス外部からはそのデリゲートを実行できなくなる。
  - 内部でのみ実行される
- これを使用することで、Eventを使用していない側からイベントが発火するのを防ぐことができる
