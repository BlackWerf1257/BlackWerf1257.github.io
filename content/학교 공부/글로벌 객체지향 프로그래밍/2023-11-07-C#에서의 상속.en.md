## Inheritance

- Creating a new class by inheriting the properties and methods of a parent class <br>

- Java and C# do not support multiple inheritance (interfaces do support multiple inheritance)

  - C++ supports multiple inheritance



## Advantages of Inheritance

* No longer need to use duplicate code in every class <br>

* Easy to modify and extend classes

<br>

## How to Use

```c#
class 부모클래스{
    //내용
}
class 자식클래스 : 부모클래스{
    //부모 클래스의 모든 상태, 내용이 전달됨
    자식 클래스만의 메소드
    자식 클래스만의 데이터
}
```
Private properties and constructors of the parent class are not created.



## Examples of Use

What you need to know when using guns, swords, and bows in an RPG game

| Gun | Sword | Bow |
| :----------------: | :----------: | :------------------: |
| Functions: Attack, Reload | Function: Attack |     Function: Attack |
| Variables: Attack Power, Magazine | Function: Attack Power | Function: Attack Power, Range |

<br/>

All three weapons have overlapping functions (Attack) and variables (Attack Power).

However, the "reload" function and the "magazine" and "range" variables are not used for all weapons.

If you add multiple weapons, you must create the same functions and variables.



**Solution:** Place common functions and variables in the superclass and implement them in the subclass by inheriting from the superclass

```c#
class Weapon{
	public void Attack() {}
    protected int power;
}

class Gun : Weapon{
    public void Reload() {}
    private int magazine;
}
class Knife : Weapon { //내용 }
class Bow : Weapon { private int reach; }
```



## Basics

* Used in methods

  <br/>

  ### How to Use

  ```c#
  base.부모메소드();
  ```

  <br/>

* Called in the constructor; it calls the parent class's constructor and then calls its own constructor

* Child constructor: **base()**



**Example**

```c#
class Base{
	public void BaseMethod() {}
}

class Derived : Base{
    public void DerivedMethod(){ base.BaseMethod(); }
}
```



## Sealed

* Meaning "sealed," it is protected from inheritance and cannot be altered or modified from the outside <br/>



* When **Sealed** is placed before a **Class**
  * Class inheritance is not allowed
* When **Sealed** is placed before a **Method**
  * Method inheritance (overriding) is not allowed



​  **Sample Code**

```c#
sealed class Parent{
	public int num;
    public Parent() { Console.WriteLine("부모 클래스의 생성자가 호출 되었습니다."); }
}

class Child : Parent{
    public int num;
    public Child(int num) {
        this.num = num;
        Console.WriteLine("자식 클래스의 생성자가 호출되었습니다.");
    }
    public void DisplayValue(){ Console.WriteLine("num의 값은 {0}입니다.", num); }
}
```
