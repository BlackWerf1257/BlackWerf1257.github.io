## Normalization

- **Decomposing** the original relation

- Data duplication occurs when the database is poorly designed

  ​  => **Update error** occurred



### Update Issues

#### Types

- **Insertion errors**
- **Editing errors**
- **Deletion errors**



#### Example

**Product Table**

| Product ID | Product Name | Product Location | Manufacturer | Category     | **Purchaser ID** | Purchaser Name |
| -------- | -------- | -------- | ------ | -------- | -------------------- | ------------------ |
| A-1234   | OO RC Car  | 3F-A12   | A | Toys     | DWA-32SV2 | Kim OO |
| C-2825   | AA Pencil Case  | 3F-F32   | B | Stationery     | AWF-34AC2 | Park XX |
| G-4326   | XX Robot  | 3F-A54   | A | Toys     | DWA-32SV2 | Kim OO |
| Y-3253   | OX TV    | 2F-G43   | C | Electronics | RSD-64BR7 | Yoo OX |

- This table uses **'Product ID' and 'Purchaser ID' as its primary key**.
- The problem with this table is that it contains both user information and product prices.



##### Insertion Error

- When adding a user's name to this table, you cannot add only the user ID and user name, which are the primary keys.

  - Since there are two primary keys in the table, unnecessary values such as "Product Name" must also be included

- This phenomenon, where unnecessary data is added when inserting specific data, is called insertion distortion.

  

##### Edits and above

- If you change the name of a user named Kim OO in a specific tuple within a table, you must also change that user’s name in all other tuples.
  - The issue that arises when data is modified in only one tuple rather than in all tuples is called a data inconsistency. 



##### Deletion and Beyond

- If a specific product is deleted from this table due to being out of stock or other reasons, the buyer’s ID and name are also deleted from this table.
  - Although user information is necessary, it is contained within the same tuple and is therefore deleted along with it.
- This phenomenon, where data that is not intended for deletion is also deleted when a specific tuple is removed, is called a deletion anomaly.



### Function Dependencies

- The Core of Normalization Theory
- A → B    // A determines B (B is functionally dependent on A)



- Total function dependence

  - (A1, A2) → B // A1 and A2 together determine B

  

- Partial functional dependence 

  - A1 → B // Avoid using if possible



### Transitive (Functional) Dependency
