## ポインタ

#### 使い方

- **データ型 *変数名;**

- **&変数：変数のアドレス**
- **変数：変数の値**

- 例

  ```c++
  char v = 10;
  char *p;
  p = &v;
  cout << v;
  cout << p;
  ```

<br/>



##### 特徴

- 一般的な変数とは異なり、アドレスを格納する

  

#### ポインタと配列の関係

- ポインタ変数に1を加える

  - ポインタ変数の型と同じサイズだけ増加する

- 配列の変数

  - 配列の変数は、配列の最初の要素のアドレスを保持する

  - *(p+1) = p[1]

- ポインタと配列は、同じ構文でデータにアクセスできる





## 手続き型プログラミング

#### 種類

- C、Fortran...

#### 特徴

- 作業の順序に基づいたプログラミング
- 機能重視
- 直感的で、公平かつ、定められたルーチンに基づくプログラムの作成に適している



#### メリット

- コンピュータの処理構造と類似している
  - 実行速度が速い
- プログラムの流れを容易に追跡できる



#### 欠点

- 各コード間の密接な関連性が高い
  - 保守の難易度が上がる
    - 新機能やデータの追加が困難であり
    - 一部が故障すると、システム全体の故障に拡大する
- 実行順序が固定されている
  - コードの順序が変わった場合、同じ結果を保証することが難しい
- デバッグが困難である



## 構造化プログラミング

#### 種類

- C++、C#、Java、Python
- データに関連する動作のグループをオブジェクトとして定義
- データ中心であり、安定性が高い
- データや動作が変更された際のメンテナンスが容易
- データに関連する動作でエラーが発生した際、デバッグが容易



#### 特徴

- カプセル化
  - プロパティとメソッドをひとまとめにし、その内部内容へのアクセスを外部から制限する

- 抽象化
  - オブジェクトの共通的な属性や機能のみを実装すること

- 継承
  - 親クラスの属性やメソッドを子クラスで完全に実装すること

- 多型
  - 1つのオブジェクトが複数の型を持つことができること
    - オーバーライドとオーバーライディングが存在する




#### メリット

- モジュール化およびカプセル化 -> 保守が容易
- オブジェクト指向言語
  - 現実世界との類似性により、コードの記述が容易
- オブジェクト自体がひとつのプログラム
  - 再利用が容易



#### 欠点

- 比較的処理速度が遅く、多くのメモリを消費する
- コード設計に、手続き型プログラミングに比べて多くの時間がかかる





## クラス

### 定義

- オブジェクト生成のために定義された型
- クラスはオブジェクトではなく、実体でもない
- メンバ変数／関数の宣言



#### 宣言方法

```c++
class 클래스이름{
    public: 접근제한자(private 등 다른 접근 제한자 사용 가능)
        변수타입 변수명;

```

#### 実装方法

```c++
함수의 리턴타입 클래스이름 :: 멤버함수명과 매개변수(){

```

#### オブジェクト

- 生成時にクラスの構造をそのまま引き継ぐ
- メンバー変数と関数で構成される
- メモリ上に生成される
  - インスタンスとも呼ばれる
- 1つのクラスから複数のオブジェクトを生成できる
- オブジェクトはそれぞれ独立した領域に生成される

<br/>

#### メンバー変数とメンバー関数

##### メンバー変数

- クラス内に存在するデータ
  - 例）速度、前進／後退

##### メンバー関数（メソッド）

 - メンバー変数にアクセスできる
 - オブジェクトの動作を定義する
   - 走る、停止する、左右への方向転換



##### 例 1

```c++
class Circle {
public:
    int radius;
    double getArea();
};

double Circle:: getArea() {
        return 3.14 * radius * radius;
    };

int main(){
    Circle donut;
    donut.radius = 10;
    double area = donut.getArea();
    cout << "크기: " << area << endl;

    Circle pizza;
    pizza.radius = 30;
    double area2 = pizza.getArea();
    cout << "크기: " << area2 << endl;
}
```

**結果**

```
크기: 314
크기: 2826
```

## コンストラクタ

- オブジェクトが生成される時点で自動的に呼び出されるメンバ関数
- クラス名と同じ名前のメンバ関数

#### 例

```c++
class Circle{
    Circle();
    Circle(int radius);
}
Circle::Circle(){ }
Circle::Circle(int radius) {}
```



#### 目的

- オブジェクト生成時に、オブジェクトに必要な初期化を行うため
  - メンバ変数の値の初期化、メモリの割り当て、ファイルのオープン、ネットワーク接続...

#### 特徴

- **作成者名**

  - 必ずクラス名と同じでなければならない

  <br/>

- **コンストラクタは戻り値の型を宣言しない**

  - 戻り値なし（Void型も不可）

  <br/>

- **オブジェクト生成時に1回のみ呼び出す**

  - 自動的に呼び出され、手動で呼び出すことはできない
  - 各オブジェクトごとにコンストラクタが実行される

  <br/>

- **作成者は重複可能です**

  - 1つのクラス内に複数のコンストラクタを定義できる
  - 重複するコンストラクタのうち、1つだけが実行される

  <br/>

- **生成されていない場合、デフォルトコンストラクタが自動的に生成されます**

  - デフォルトコンストラクタ - 引数なしのコンストラクタ
  - コンパイラによって自動的に生成される

#### デフォルトコンストラクタ

##### 特徴

- オブジェクト生成時に1回呼び出される

- クラスにコンストラクタがない場合、コンパイラが自動的に生成する
  - クラス内に引数付きのコンストラクタしか存在しない場合、引数なしでオブジェクトを作成しようとするとエラーが発生する
- 引数がない

###### サンプルコード

~~~c++
class Rectangle
{
public:
    int width, height;
    Rectangle();
    Rectangle(int w, int h);
    Rectangle(int length);
    bool isSquare();
};
Rectangle::Rectangle() { width = height = 1; } //Rect1から呼び出される
Rectangle::Rectangle(int w, int h) { width = w, height = h; } //Rect2から呼び出される
Rectangle::Rectangle(int length) { width = height = length; } //Rect3から呼び出される
bool Rectangle::isSquare() {
    if (width == height) return true;
    else return false;
}


int main(){
    Rectangle rect1, rect2(3, 5), rect3(3);
if (rect1.isSquare()) cout << "rect1は正方形" << endl;
if (rect2.isSquare()) cout << "rect2は正方形" << endl;
if (rect3.isSquare()) cout << "rect3は正方形" << endl;
}
~~~

**結果**

~~~
rect1は正方形
rect3は正方形
~~~
