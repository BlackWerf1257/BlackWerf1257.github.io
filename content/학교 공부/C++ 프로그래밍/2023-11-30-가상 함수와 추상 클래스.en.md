## Dynamic Binding

### Features

- Executing an overridden function of a derived class via a pointer to the base class while the program is running
- Declaring a base class function as virtual
- Cannot be used with overloading (can only be used with overriding)



### Overriding Virtual Functions

#### Features

- The function name, parameter types, number of parameters, and return type must match

- The keyword "virtual" can be omitted in derived classes (since it is inherited)

**Example**

```c++
class Base{ public: virtual void f(); };
class Derived : public Base { public: virtual void f(); } //vrtual void f()와 동일한 선언으로 상속되었으므로 virtual 생략가능
```

**Example Code 1**

```c++
class Base {
    public: 
    virtual void f() 
    { cout << "Base::f() Called" << endl; }; 
};
class Derived : public Base {
    public: 
    void f(){
        cout << "Derived::f() Called" << endl; 
    }; 
};
class AgainDerived : public Derived { 
    public: 
    void f() { 
        cout << "AgainDerived:f() called" << endl; 
    } 
};

int main()
{
    AgainDerived a;
    Base* bp;
    Derived *dp;
    AgainDerived* ap;
    
    bp = dp = ap = &a;

    bp->f();
    dp->f();
    ap->f();
}

```

**Output**

```
AgainDerived:f() called
AgainDerived:f() called
AgainDerived:f() called
```



### The Scope Specifier Operator (::) in Overriding

- Scope-specifying operator **'::'**
  - Indicates static binding
  - Calls a virtual function of a base class via static binding in the form **BaseClass::virtualFunction()**
    - Example: Shape::draw();



## Virtual Annihilator

- Declare the destructor using the `virtual` keyword
- Dynamic binding occurs when the destructor is called
  - The destructor of the derived class takes precedence



#### In the case of a virtual annihilator

```c++
class Base4 {
public:
    virtual ~Base4() { cout << "Called from Base" << endl; };
};
class Derived4 : public Base4 {
public:
    ~Derived4() { cout << "Called from Derived" << endl; };
};


int main(){
    Base4* pT = new Derived4();
	delete pT;
}
```

**Output**

```
Called from Derived
Called from Base
```



1. Call the ~Base4() destructor
2. Execute ~Derived4()
3. Execute ~Base4()



#### If it is not a virtual destructor


## Overriding Virtual Functions

- The role of the base class
  - The interface of the virtual function `draw`
- Overriding virtual functions in derived classes
  - Implementing the `draw` function differently in each derived class (polymorphism)



```c++
//Shape.h
class Shape {
	Shape* next;
protected:
	virtual void draw();
public:
	Shape() { next = NULL; }
	virtual ~Shape() {}
	void paint();
	Shape* add(Shape* p);
	Shape* getNext() { return next; }
};

//Shape.cpp
#include <iostream>
#include "Shape.h"
using namespace std;

void Shape::paint() {
	draw();
}

void Shape::draw() {
	cout << "--Shape--" << endl;
}

Shape* Shape::add(Shape* p) {
	this->next = p;
	return p;
}
```

## Applications of Virtual Functions

- Example of dynamic binding using virtual functions
  - Accessing a derived class via a pointer to the base class
  - The types of pStart, pLast, and p are Shape*
  - Accessing derived objects that inherit from Shape by traversing the linked list
  - Calling the override function draw in the derived object via the call to p->paint()



```c++
#include "Shape.h"
#include "Circle.h"
#include "Rect.h"
#include "Line.h"

int main(){
    Shape* pStart = NULL;
Shape* pLast;

pStart = new Circle();
pLast = pStart;
pLast = pLast->add(new Rect());
pLast = pLast->add(new Circle());
pLast = pLast->add(new Line());
pLast = pLast->add(new Rect());

//현재 연결된 모든 도형 그리기
Shape* p = pStart;
while (p != NULL) {
    p->paint();
    p = p->getNext();
}
//현재 연결된 모든 도형 삭제
p = pStart;
while (p != NULL) {
    Shape* q = p->getNext(); //다음 도형 주소 기억
    delete p; //기본 클래스의 가상 소멸자 호출
    p = q; //다음 도형 주소를 p에 저장
	}
}
```

**Output**

```
Circle
Rectangle
Circle
Line
Rectangle
```





## Pure virtual functions

- Purpose of virtual functions in base classes

  - The role of indicating which functions should be overridden in a derived class

    - The purpose of writing the code is X

  - No need to implement the virtual functions of the base class

    

- Pure virtual function

  - Pure Virtual Function

  - **Virtual member functions with only a declaration, where the function code is X**

    - Eliminates unnecessary code duplication

      

  ### How to Declare

```c++

public:
	 	virtual void draw() = 0; //순수 가상 함수 선언
```

## Abstract Class

### Features

- A class with one or more pure virtual functions
- Object creation is not possible due to pure virtual functions
  - Used for inheritance and design purposes
- Pointers can be declared
  - Can be utilized with dynamic binding



### Inheritance

- If you simply inherit from an abstract class, the derived class automatically becomes an abstract class.

### Implementation

- Inherit from an abstract class and override a pure virtual function

  - A derived class is an abstract class X

  <br/>

  #### Examples of Incorrect Implementations

```c++
class Shape {
public:
    virtual void draw() = 0;
};
class Circle : public Shape {
public:
    string toString() { return "Circle 객체"; }
};

Shape shape;
Circle waffle;
```

- Both Shape and Circle are abstract classes

  - Therefore, both `shape` and `waffle` result in an object creation error.

    **Error Name**

```
심각도	코드	설명	프로젝트	파일	줄	비표시 오류(Suppression) 상태
오류(활성)	E0322	추상 클래스 형식 "Circle"의 개체를 사용할 수 없습니다.

```



#### Example of Correct Implementation

```c++
class Shape { //추상 클래스
public:
    virtual void draw() = 0;
};
class Circle : public Shape { //추상 클래스가 아님
public:
    virtual void draw() { // 순수 가상 함수 오버라이딩
        cout << "Circle";
    }
    string toString() { return "Circle 객체" };
};

Shape shape;   //추상 클래스이므로 객체 생성 오류 발생
Circle waffle; //순수 가상함수를 오버라이드 했으므로 정상적으로 객체가 생성됨
```





**Example**

```c++
class Calculator {
    void input() {
        cout << "정수 2개 입력 >>";
        cin >> a >> b;
    };

protected:
    int a, b;
    virtual int calc(int a, int b) = 0; //두 정수의 합 리턴
public:
    void run() {
        input();
        cout << "계산된 값 : " << calc(a, b) << endl;
    }
};
class Adder : public Calculator {
protected:
    int calc(int a, int b) { //순수 가상 함수 구현
        return a + b;
    }
};
class Subtractor : public Calculator {
protected:
    int calc(int a, int b) { //순수 가상 함수 구현
        return a - b;
    }
};


int main(){
	Adder adder;
Subtractor subtractor;
adder.run();
subtractor.run();    
}
```

**Output**

```
정수 2개 입력 >>  //5 7로 진행
계산된 값 : 12
정수 2개 입력 ->> // 7 5로 진행
계산된 값 : 2
```

- Proceed in the order **'+'** -> **'-'**



## Generalization Using Templates

### Generalization / Generic Functions

- How to create a framework that can be used regardless of the variable types in functions and classes
- How to group the entire set into a single unit

### Template

- Tools in C++ for generalization
- Using the **template** keyword



### Declaration

```c++
template <class T>
    or
template <typename T>
    
    
```



## Instantiation of Generic Functions

### Specification

- The compiler generates code for a function specific to a particular data type from a template function

**Example**

```c++
template <class T>
    void swap(T& a, T& b){
    T tmp;
    tmp = a;
    a = b;
    b = tmp;
}

int main(){
	int a = 4, b = 5;
    swap(a,b);
}
```





## Pros and Cons of Templates

###  Advantages

- Reuse of function code
  - Increased software productivity and usability

### Disadvantages

- Prone to porting issues
  - May not be supported by certain compilers
- Insufficient compile error messages, making debugging very difficult

## Template Examples


### Example 2
