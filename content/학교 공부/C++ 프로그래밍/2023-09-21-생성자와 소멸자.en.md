## The Destroyer

### Meaning

A function that is automatically called when an object is destroyed

<br/>

### Features

- Cannot be called arbitrarily

- Cannot exist in more than one instance (polymorphism is not possible)

- When an object is destroyed, it performs tasks such as freeing allocated memory, saving and closing files, and closing network connections

- Prefix the class name with **~**

  Example

  ~~~c++
  class Circle{
 Circle();
 ~Circle(); 
 }
  Circle::~Circle();
  ~~~

  

  

- No return type

- If no destructor is declared, a default destructor is automatically generated.



### Execution Order of Constructors and Destructors

#### Classification by Object Creation Location

##### Regional Objects

- Objects declared within a function are destroyed when the function ends

<br/>

##### Global Objects

- Objects declared outside a function are destroyed when the program terminates

<br/>

#### Object Creation Order

- Global objects: Created in the order they are declared in the program
- Local objects: Created in order when the function is called

<br/>

#### Object Destruction Order

- When a function ends, local objects are destroyed in the reverse order of their creation.
- When the program ends, global objects are destroyed in the reverse order of their creation.

<br/>

#### Objects dynamically created using the `new` keyword

- An object is created the moment `new` is executed
- An object is destroyed the moment the `delete` operator is executed

**Sample Code**

```c++
class Circle {
	public:
		int radius;

		Circle();
		Circle(int r);
		~Circle(); //소멸자
		double getArea();
};
Circle::Circle() {
	radius = 5;
	cout << "지름이 " << radius * 2 << "인 원 생성" << endl;
}
Circle::Circle(int r) {
	radius = r;
	cout << "지름이 " << r * 2 << "인 원 생성" << endl;
}
Circle::~Circle() {
	cout << "지름이 " << radius * 2 << "인 원 소멸" << endl;
}
double Circle::getArea() {
	return 3.14 * radius * radius;
}

Circle globalDonut(1000);
Circle globalPizza(2000);
void func() {
	Circle fDonut(100);
	Circle fPizza(200);
}
	
	
	
int main(){
	Circle mainCircle1;
    Circle mainCircle2(10);
    func();
}
```

**Execution Order in the Sample Code**

1. **Program loading** - Create globalDonut and globalPizza objects
2. **Execute main function** - Create mainCircle1 and mainCircle2 objects
3. **Execute the func function** - Create fDonut and fPizza objects
4. **Exit the func function** - Destroy the fDonut and fPizza objects
5. **Exit the main function** - Destroy the mainCircle1 and mainCircle2 objects
6. **Exit the program** - Destroy the globalDonut and globalPizza objects

## Access Modifiers

### Types

- **public** - Accessible from external functions
- **private (default)** - Accessible only by member functions of the same class
- **protected** - Accessible only by the class itself and its subclasses

### Features

- Supports object encapsulation
- Specifies access permissions for members

<br/>

## Inline Functions

### Issues with the Function Call Process:

- Frequent calls cause unnecessary overhead issues

  Save registers to the stack -> Call and execute the function -> Restore registers saved on the stack -> Return from the function

  Registers - Return address, local variable values, parameters

### How to Use

- Using the **inline** keyword

- Automatically inserted into the code during the compilation process by the preprocessor

- Sample code

  ~~~c++
  inline int divide(int x) {
 return (x/2);
  }
 
 int main() {
    int sum = 0;
 
 for(int i=1; i<=3; i++) {
 if(divide(i)) //(x/2) 호출
  		{
 cout << i << " 호출 " << endl;
 sum += i;
 }
    }
    cout << "----" << endl;
    cout << sum;
  }
  ~~~

  **Results**

  ~~~
  Call 2 
  Call 3 
  ----
  5
  ~~~

  <br/>

### Advantages

- Reduces the overhead of frequently called functions -> Improved performance

### Disadvantages

- Memory overhead caused by an increase in the size of the executable program

### Automatic Inline Functions

- Functions that the compiler inlines even without using the `inline` keyword

- Example)

  ~~~c++
  class Circle{
  private:
 int radius;
  public:
 Circle() { radius = 1; } //자동 인라인 함수
 Circle(int r);
 double getArea() { return 3.14 * radius * radius; }
  };
  ~~~

## Structures

### Features

- Similar to classes (inheritance, members, access modifiers)
- The default access modifier is **public**
- Compatible with C

### How to Use

~~~c++
struct struct_name {
  type member_variable1;
    ...
  type member_variable_n;
}
~~~

<br/>

**Sample Code**

## File Separation

### Why Use It

**Separating class declaration and implementation**

**Separation of Declaration and Implementation**

- Efficient for reusing and modifying classes
- Class declarations are managed in header files
- Class implementations are managed in CPP files

<br/>

**Efficient for project management**

- Facilitates the division of development tasks
- Manages detailed code at the class level
- Minimizes the impact of changes during collaboration

<br/>

### How to Use External Files

**#include "filename.h"**

<br/>

### Issues Caused by Duplicate Headers

Issues caused by duplicate header declarations can be resolved by using the following preprocessor directive:

~~~c++
#ifndef header_name
#define header_name
~~~

## Object Pointers

- A variable that holds the address of an object

- When accessing a member via a pointer, use the format **p->member**

  <br/>

### How to Use

- Declaring a pointer variable
  - **ClassName *variable**
- Assigning an address to a pointer variable
  - **&objectName**
- Accessing members via an object pointer
  - **->**

<br/>

**Sample Code**

~~~c++
class CirclePointer {
  int radius;
  public:
    CirclePointer() { radius = 1; }
    CirclePointer(int r) { radius = r; }
    double getArea();
};
double CirclePointer::getArea() {
  return 3.14 * radius * radius;
  }


int main(){
//객체 포인터
CirclePointer donut;
CirclePointer pizza(30);

// Accessing members by object name
cout << donut.getArea() << endl;

// Accessing members via an object pointer
CirclePointer *p;
p = &donut;
cout << p->getArea() << endl; // Calls donut's getArea
cout << (*p).getArea() << endl; // Calls donut's getArea

p = &pizza;
cout << p->getArea() << endl; // Call getArea on pizza
cout << (*p).getArea() << endl; // Call getArea on pizza
}
~~~

**Results**

3.14
3.14
2826
2826
~~~
