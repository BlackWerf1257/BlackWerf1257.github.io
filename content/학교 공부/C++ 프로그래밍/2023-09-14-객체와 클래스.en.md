## Pointers

#### How to Use

- **DataType *VariableName;**

- **&variable: the address of the variable**
- **variable: the value of the variable**

- Example

  ```c++
  char v = 10;
  char *p;
  p = &v;
  cout << v;
  cout << p;
  ```

<br/>



##### Features

- Unlike regular variables, it stores an address

  

#### The Relationship Between Pointers and Arrays

- Increment a pointer variable by 1

  - Increases by the size of the pointer variable's type

- Array variables 

  - An array variable holds the address of the first element of the array

  - *(p+1) = p[1]

- Pointers and arrays allow access to data using the same syntax





## Procedural Programming

#### Types

- C, Fortran...

#### Features

- Programming based on the sequence of tasks
- Focus on functionality
- Intuitive, fair, and conducive to creating programs with fixed routines



#### Advantages

- Similar to a computer's processing architecture
  - Fast execution speed
- Easy to track the program's flow



#### Disadvantages

- High interdependence between code segments
  - Increased difficulty in maintenance
    - Difficult to add new features or data, and
    - A failure in one part can cause the entire system to fail
- Fixed execution order
  - Difficult to guarantee the same results if the order of the code is changed
- Difficult to debug



## Structured Programming

#### Types

- C++, C#, Java, Python
- Defines a group of data-related behaviors as an object
- Data-centric and highly stable
- Facilitates maintenance when data or behaviors change
- Facilitates debugging when errors occur in data-related behaviors



#### Features

- Encapsulation
  - Grouping properties and methods together and restricting access so that the actual contents cannot be accessed from the outside

- Abstraction
  - Implementing only the common attributes and functions of objects

- Inheritance
  - The process of a subclass implementing all the properties and methods of a superclass

- Polymorphism
  - The ability of a single object to have multiple types
    - Overriding and overridden methods exist




#### Advantages

- Modularity and encapsulation -> Easy to maintain
- Object-oriented languages
  - Easy to write code due to its similarity to the real world
- An object itself is a program
  - Easy to reuse



#### Disadvantages

- Relatively slow and memory-intensive
- Requires more time to design the code compared to procedural programming

## Constructor

- A member function that is automatically called when an object is created
- A member function with the same name as the class

#### Example

```c++
class Circle{
    Circle();
    Circle(int radius);
}
Circle::Circle(){ }
Circle::Circle(int radius) {}
```



#### Purpose

- To initialize the object as needed when it is created
  - Initializing member variable values, allocating memory, opening files, establishing network connections...

#### Features

- **Creator's Name**

  - Must be the same as the class name

  <br/>

- **Constructors do not declare a return type**

  - No return type (including `void`)

  <br/>

- **Called only once when creating an object**

  - Called automatically; cannot be called manually
  - The constructor is executed for each object

  <br/>

- **Creators can be duplicated**

  - A class can have multiple constructors
  - Only one of the duplicate constructors is executed

  <br/>

- **If no constructor is defined, a default constructor is automatically generated.**

  - Default constructor - Constructor with no parameters
  - Automatically generated by the compiler

#### Default Constructor

##### Features

- Called once when an object is created

- If a class does not have a constructor, the compiler generates one automatically.
  - If a class contains only constructors with parameters, an error will occur when attempting to create an object without parameters.
- No parameters

###### Sample Code

~~~c++
class Rectangle
{
public:
    int width, height;
    Rectangle();
    Rectangle(int w, int h);
    Rectangle(int length);
    bool isSquare();
};
Rectangle::Rectangle() { width = height = 1; } //Rect1에서 호출됨
Rectangle::Rectangle(int w, int h) { width = w, height = h; } //Rect2에서 호출됨
Rectangle::Rectangle(int length) { width = height = length; } //Rect3에서 호출됨
bool Rectangle::isSquare() {
    if (width == height) return true;
    else return false;
}


int main(){
    Rectangle rect1, rect2(3, 5), rect3(3);
if (rect1.isSquare()) cout << "rect1은 정사각형" << endl;
if (rect2.isSquare()) cout << "rect2은 정사각형" << endl;
if (rect3.isSquare()) cout << "rect3은 정사각형" << endl;
}
~~~

**Results**

~~~
rect1 is a square
rect3 is a square
