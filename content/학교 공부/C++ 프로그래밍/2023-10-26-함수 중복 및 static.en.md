## References

### Call by Reference

#### Declaring function parameters as reference types

##### Format

- **Variable type Function name (Variable type & Variable name)**

- Example

~~~c++
void testFunc(int &n1, int &n2);
~~~

#### When a reference call is required

- When multiple types of data are returned
  - Value or error information

<br/>

#### Returning Values Using References

- Example

  ~~~c++
  #include <iostream>
  ~~~

  ### Reference Return

  #### Function Returns in C

  - Functions must return only a value

  -  Basic data types: int, char, double…

  -  Pointer value

   <br/>

  #### Function Returns in C++

  - Functions can return references as well as values

  - Return a reference
    - Returns a reference to an existing location, such as a variable
 - Does NOT return the value of the variable

<br>

### Copying Objects

 #### Shallow Copy

- Copies **only** the pointer
- Shares the address of a dynamically allocated variable

#### Deep Copy

- Copies the memory address of the pointer
- Once copied, the object becomes an independent entity, so even if the original object changes, it has no effect

 <br/>

 ### Copy Constructor

#### Features

- The constructor called when an object is copied

- Only **one** can be declared per class (duplicate functions are not allowed)

- Has a reference parameter for the class

#### Implementation Method

~~~c++
class ClassName {
  ClassName(const ClassName& objectName); // Declaration of the copy constructor
}

//Implementation of the copy constructor
ClassName::ClassName(const ClassName& objectName);
~~~

**Example**

~~~c++
#include <iostream>
using namespace std;

class Square {
private:
    int length;
public:
  Square(const Square& s);
    Square() :Square(5) {};
    Square(int l) { length = l; }
    int GetArea() { return length * length; }
};

// Copy constructor
Square::Square(const Square& s) {
    this->length = s.length;
    cout << "Copy constructor executed" << endl;
}

int main() {
    Square b1;
    Square b2(b1);

    cout << "Area of b1>>" << b1.GetArea() << endl;
    cout << "Area of b2>>" << b2.GetArea() << endl;
}
~~~

**Results**

~~~
Copy constructor executed
Area of b1 >> 25
Area of b2 >> 25
~~~

### Polymorphism

#### Function Duplication

- Functions with the same name can be used

- **But** there must be a difference in the type or number of variables

- The return type has nothing to do with function overloading

- Using functions with the same name -> The advantage is that there is no need to distinguish between function names

 

#### Overloading

- Duplicate functions

- Example

  ~~~C++
  //Correct example
  int(int a, int b) { return a+b;}
  int(char a, char b) {return a+b;}
  
  // Incorrect examples
  double(int a, int b) { return a+b;} // The return type is irrelevant to function overloading
  int(char a, char b) {return a+b;}
  ~~~

  

#### Overriding

- Function duplication among inherited objects

- Destructors cannot be overloaded

- Does not retain parameters

<br/>

#### Default parameters

##### Features

-  The default value assigned to a parameter in the called function when no value is passed

##### Example

 ~~~c++
 //Declaration section
 void TestVoid(int id=0, string name="Test"){
 cout << "ID>>" << id << endl;

 }
 
 //Call section
 TestVoid();
 TestVoid(2021, "Kim");


//Result

 Name>>Test

 ~~~

##### Restrictions

-  Cannot be declared before general parameters

- Must be declared after the general parameters
