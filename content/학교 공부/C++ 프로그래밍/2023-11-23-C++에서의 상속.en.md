## Inheritance

### Accessing Inherited Objects

```c++
class Shape {
    int x, y; //한 점 (x,y)의 좌표값
    int type; //0:미지정, 1:원 2:사각형
public:
    Shape() { type = 0; }
    void set(int x, int y) {
        this->x = x;
        this->y = y;
    };
    void showShape() {
        cout << "(" << x << "," << y << ")" << endl;
    }
};
    class Circle : public Shape {
        string color;
    public:
        void setColor(string color) { 
            this->color = color; 
        }
        void showCircle();
    };
    void Circle::showCircle() {
        cout << color << ":";
        showShape(); //Shape의 showShape() 호출
    }
    
int main()
{
    Circle a;
    a.set(1, 2); //기본 클래스 Shape의 set(int x, int y) 호출
    a.showShape(); //기본 클래스 Shape의 showShape() 호출
    a.setColor("Red"); //파생 클래스 Circle의 setColor(string color) 호출
    a.showCircle(); //파생 클래스의 showCircle();
}
    
```

**Output**

```
(1,2)
Red:(1,2)
```

<br/>

- The Circle class inherits from the Shape class.

  

  - In `a`, an object of the derived class, `set()` and `showShape()` call the functions inherited from the parent class, and

  - Since `setColor(string color)` and `showCircle()` are implemented in the derived class, they call functions within the derived class.

    <br/>

### Access Modifiers in Inheritance

- #### private

  - Accessible only within the class where it is declared

   - Cannot be accessed directly from a derived class

- #### public

  - All external functions are accessible from the declared class or any external class
  - Derived classes can access the public members of the base class

- #### protected

  - Accessible from the declaring class
  - Access permitted only from derived classes

#### Example

```c++

#include <iostream>
```

### Calling Constructors and Destructors

- #### Execution of the constructor

  - Execute the base class constructor first

- #### Execution of the Destroyer

  - The destructor of the derived class is executed first

```c++
class Parent_Constructor {
public:
    Parent_Constructor() {
        cout << "부모 클래스에서 생성자가 호출 되었습니다." << endl;
    }
    ~Parent_Constructor() {
        cout << "부모 클래스에서 소멸자가 호출 되었습니다." << endl;
    }
};
class Child_Constructor : public Parent_Constructor {
public:
    Child_Constructor() {
        cout << "자식 클래스에서 생성자가 호출 되었습니다." << endl;
    }
    ~Child_Constructor() {
        cout << "자식 클래스에서 소멸자가 호출 되었습니다." << endl;
    }
};


int main(){
    Child_Constructor cc;
}
```

**Output**

```
부모 클래스에서 생성자가 호출 되었습니다.
자식 클래스에서 생성자가 호출 되었습니다.
자식 클래스에서 소멸자가 호출 되었습니다.
부모 클래스에서 소멸자가 호출 되었습니다.
```

- As explained above, when both the parent class and the child class have constructors, the constructors are executed in the order of parent class first, then child class.
- When both the parent class and the child class have destructors, the destructors are executed in the reverse order: child class first, then parent class.

### Designating an Heir

- #### Designated Heir

  - ##### public

    - Inherits the protected and public member variables of the base class **identically**

  - ##### private

    - Inherit the protected and public members of the base class as **private**

  - ##### protected

    - Inherit the protected and public members of the base class as **protected**

```c++
    int res = 0;
    switch (op){
```

#### Solving the Problem of Multiple Inheritance Using Virtual

- Through virtual inheritance, you can configure the system so that members of the base class are created only once.

```c++
class Animal {
public:
    int lifetime;
};
class eagle : virtual public Animal {
public:
    int wing;
};
class lion : virtual public  Animal{
public:
    int tooth;
};
class kimera : public eagle, public lion {
public:
    int leg;
};


int main(){
    kimera wild;
    wild.wing = 10;
    wild.tooth = 20;
    wild.lifetime = 30;
}
```



### Overriding Functions in Derived Classes

```c++
class Base {
public:
    void f() { cout << "Base::f() called" << endl; }
};

class Derived : public Base {
public:
    void f() { cout << "Derived::f() called" << endl; }
};


int main(){
	Derived d, * pDer;
    pDer = &d;
    pDer->f(); //Derived::f() 호출

    Base* pBase;
    pBase = pDer; //업캐스팅
    pBase->f(); //Base::f() 호출
}
```

**Output**

```
Derived::f() called
Base::f() called
```



### Virtual Functions and Overriding

#### Virtual Functions

- Declared as **Virtual**
  - Call binding for the function is determined at runtime (dynamic binding)
- Overriding
  - Declaring a function in a derived class with the same name as a function in the base class
    - Replaces the implementation of the base class function
    - A feature for polymorphism
    - Overriding a virtual function enables dynamic binding

```c++
class Base2 { 
    public: 
        virtual void f() {  //가상 함수
            cout << "Base::f() Called" << endl; 
        } 
};
class Derived2 : public Base2 { 
public: 
    virtual void f() {  //오버라이딩
        cout << "Derived::f() called" << endl; 
    } 
};


int main(){
    Derived2 d, * pDer2;
    pDer2 = &d;
    pDer2->f(); //Derived::f() 호출

    Base2* pBase2;
    pBase2 = pDer2; //업캐스팅
    pBase2->f(); //Base::f() 호출
}
```

**Output**

```
Derived::f() called
Derived::f() called
```

<br/>

- Static binding

  - Determining function calls at compile time

- Dynamic binding

  - Determining function calls at runtime

  ![Image Alt text]({{link}}/assets/img/Cpp/11th/Binding_Type.png )

**Example**

```C++
class Base2 { 
    public: 
        virtual void f() { 
            cout << "Base::f() Called" << endl; 
        } 
};
class Derived2 : public Base2 { 
public: 
    virtual void f() { 
        cout << "Derived::f() called" << endl; 
    } 
};


int main(){
    Derived2 d, * pder; //Base 멤버의 void f()는 호출되지 않음
    pder = &d;
    pder->f(); //derived::f() 호출

    Base2* pbase;
    pbase = pder; //업 캐스팅
    pbase->f(); // 동적 바인딩 발생 derived::f() 실행
}
```

**Output**

```
Derived::f() called
Derived::f() called
```
