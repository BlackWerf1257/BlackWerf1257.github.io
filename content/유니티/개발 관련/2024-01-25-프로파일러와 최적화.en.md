### GPU Profiling

- As of the latest version of the documentation (Editor 2022.1) listed below, this feature is only available in DirectX 11 and 12.
- You can open the profiling window from the Windows > Analysis > Profiler menu.

#### Meaning

- A profiler that shows which parts of the game used the GPU and for how long

- Unit: ms

If you look closely at the image above, it shows where the DrawCall was made to the GPU, as well as the number of calls and the elapsed time.

#### Related Concepts

- FillRate = Number of pixels on the screen × Fragment shader complexity × Overdraw

  - Number of pixels on the screen: Screen resolution
  - Fragment shader complexity: A measure of how resource-intensive the shader is
  - Overdraw: A measure of how much pixels overlap when rendered

  
  
  #### Advantages
 
 - Since there is no need to build a player, it is advantageous for fast profiling
 
 #### Disadvantages
 
 - Overhead may affect the accuracy of the results

### Texture Optimization

- When using images in Unity, they are converted into a format compatible with the GPU regardless of their original format, so as long as the texture settings are configured correctly, there should be no issues.

#### Format

##### PVRTC

- Size: 4 bpp, 2 bpp (select the number of bits per pixel)

- Primarily used on iPhones

- Principle: <br/>

  1) Reduce the size of the original image by converting it into 4x4 blocks <br/>

  2) Select two representative colors <br/>

  3) Processed by combining and transferring to Modulation Data at a 1:1 resolution <br/>

- Cons: <br/>
  - Since images are rendered at 4x4 pixels, a blurring effect occurs that makes the image look pixelated; therefore, this is not recommended for retro-style games.

##### ETC

- Size:

  - ETC1: 4bpp

  - ETC2: 4, 8pp

- Principle:

  - Similar to PVRTC, it processes data in blocks, but it composites the image by separating it into **color** and **luminance** components
  - The color component is rendered at a lower resolution, while the luminance component is rendered at full resolution, and the two are then composited

- Cons:

  - Colors may appear blotchy in blocks
    - This appears less pronounced in ETC2 than in ETC1

  <br/><br/>

  There are various other file extensions, but since they are explained in detail in the official documentation, I will provide a link instead.

  <a href="https://docs.unity3d.com/kr/2021.2/Manual/class-TextureImporterOverride.html" target="Platform Recommended Format"  rel="noopener noreferrer">플랫폼별 권장 포맷</a>

  <br/>

  #### Resolution
 
 - Since it is processed in 4x4 blocks, the size must be a power of 2 (POT)
  - Available sizes: 4, 8, 16, 32, 64, 128, 256, 512, 1024, 2048...
    - If a value other than these is entered, Unity automatically scales up the size for processing, resulting in wasted space
  - With ASTC, you can freely select block sizes such as 5x5 or 12x12

### Optimization Tips

#### Memory Management

- Memory optimization becomes increasingly important, especially for mobile games

  - When using ETC[^1]2 compression, verify that the device supports ES 3.0 or higher

    - When using ETC2 in OpenGL ES 2.0, the data is decompressed, causing memory usage to increase again (4 MB → 16 MB)

    - If your target is an ES2.0 device, enable the **Compress using ETC1** option in Build Settings

      (Supported only in Sprite&UI; a Packing Tag is required)

    <br/>

  - You can save memory by specifying the Load Type for audio
    - Decompress On Load: Offers fast response times, but since the file is decompressed and played after loading, there is no guarantee that it will fit into memory at its original size
 
 ​  Decompression uses more than 10 times the memory of keeping the file compressed, so this method is only suitable for small files
    
    - Compress In Memory: Keeps the file compressed in memory and decompresses it during playback
 
 ​  Should only be used for large files that consume significant memory during decompression
 
 - Streaming: A method that decodes sound immediately
 
 ​  Reads data incrementally from disk and stores compressed data in a buffer to minimize memory usage
    
    <br/>
 
 - Since Unity does not have a feature to verify if files are identical, be careful to avoid duplicate files (fonts, textures, etc.)
    - You can check for duplicate files using the Memory Profiler
 
 - When using Meshes, ensure the **Read/Write Enable** option is turned off
    - This option determines whether scripts can access the mesh; since it loads the mesh into both GPU and CPU memory, turn it off if not needed
 
 [^1]: Android's standard texture format
 
 <br/>

#### Programming and Code Architecture

- Understand the call order of event functions such as Awake, OnEnable, and Update
  - <a href="https://docs.unity3d.com/kr/2022.3/Manual/ExecutionOrder.html" target="Event Function Call Order" rel="noopener noreferrer">Event Function Call Order</a>

- Appropriate TargetFrameRate
  - Set it according to genre and situation
  - Use Adaptive Performance
- Use asynchronous loading
  - You can improve performance by handling tasks such as asset loading asynchronously

- Remove unused Monobehaviors methods
  - Start, Update, FixedUpdate...
- Use the manager pattern instead of Update

<br/>

- Minimize high-cost API calls
  - GameObject.Find()
    - Can be replaced with transform.find, specifying via the Inspector, or FindWithTag...
  - GameObject.GetComponent
    - TryGetComponent() has a lower call cost
- Be cautious with string operations
- Be cautious with Debug.Log()
  - Use the Debug class, specify conditions with Conditional, or remove the code after use to exclude it from builds
- Be cautious with allocation-intensive APIs
  - APIs ending in "s" allocate data as arrays, so they have high call costs
- Be cautious when parsing static data
  - JSON/XML: ScriptableObject
